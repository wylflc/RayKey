# A股选股-估值-量价操作流程

## 1. 目标

本流程用于固定“全市场质量筛选 -> 核心公司估值排雷 -> 每日右侧量价触发 -> 买入候选输出”的可重复执行体系。

核心原则：

1. 业务质量决定是否值得长期跟踪。
2. 估值分析用于排除过度透支的股票，而不是只寻找显著低估股票。
3. 量价信号只在高质量、估值合格的股票池内触发。
4. 每日输出的是买入候选，不是自动买入指令。
5. 每一次业务质量、估值、量价、买入候选、移出或降级结论，必须同步写入 `data/processed/a_share_workflow_decision_log.csv`，保留简要信息来源、结论、结论时间、输入文件、输出文件、数据口径和判断理由。

## 3. 核心概念

**值得关注公司**：
通过业务质量筛选后，值得持续跟踪的上市公司。

**临界待定公司**：
两种情形之一：缺少可靠资料、暂时无法明确其实际水平且无明显负面（含次新股披露不足）；或可判断但当前缺乏持久优势，但所处行业并非结构性绝望，未来可能因新产品、客户验证、技术路线、行业格局、资产重组或经营改善而重新获得复核资格。临界待定公司不进入每日量价扫描。

**垃圾公司**：
永久业务排除公司。只给造假/治理灾难或结构性绝望行业；公司层面“当前无护城河但行业并非绝望”属于临界待定，不判垃圾公司。进入该类后，后续季度不再分析其业务是否改善，不再进入质量分层、估值排雷和每日量价扫描，只维护证券基础信息。

**质量分层**：
在第一轮三类初筛之后，只对 `worth_attention`（值得关注）公司执行。只根据护城河、业务类型、竞争优势可持续性、技术壁垒、资本复制难度、市场地位、现金流质量和不可替代性分为L1-L5，不考虑当前股价。L1-L4 是保留在值得关注内的质量档；L5 不是保留档，而是“不再值得长期关注”的判定，将该公司由 `worth_attention` 降级为 `boundary_pending`，等待未来硬触发再复核。永久排除的 `garbage` 公司只在第一轮产生，不进入质量分层，也不用L5表达。

**核心质量公司**：
质量分层为L1或L2的公司。

**核心估值合格池**：
核心质量公司中，当前估值未明显透支bull case的股票。包含估值档位为低估、较低估、中性、可接受较高估的股票，排除高估、无法估值、极端bull case已被定价的股票。

**买入候选**：
核心估值合格池中出现有效右侧量价信号的股票。

## 4. 总流程

```text
交易所最新A股证券名单
  -> 更新 data/raw/a_share_securities.csv
  -> 第一轮三类初筛
  -> 剔除垃圾公司
  -> 值得关注公司和触发复核的临界待定公司
  -> 季度质量审查
  -> A股值得关注公司列表
  -> L1-L5质量分层
  -> L1/L2核心质量公司
  -> 核心估值合格池
  -> 每日量价扫描
  -> 买入候选输出
  -> 人工核对公告、研报和当日催化
  -> 决定是否建仓、等待回踩或放弃
  -> 建仓后写入持仓清单
  -> 每日持仓监控与卖出扫描
  -> 持仓动作：持有/加仓资格/减仓/止盈/清仓/风险预警
```

## 5. 阶段一：季度全市场质量审查

### 5.1 执行频率

每个季度报告披露周期后执行一次完整更新；在财报季内，按单家公司报告披露进度滚动更新。

### 5.2 输入

1. `data/raw/a_share_securities.csv`
2. 上一版 `data/processed/a_share_attention_triage.csv`
3. 上一版 `data/processed/a_share_watchlist_quality_tiers.csv`
4. 公司最新年报、半年报、季报、业绩预告、业绩快报
5. 公司公告、交易所问询函、投资者关系材料
6. 权威研报和专业机构研究材料
7. 同行业历史校准文件和既有L1/L2/L3参考公司

### 5.3 季度开始先更新A股证券名单

每个季度开始分析前，先基于交易所公开名单刷新A股证券基础名单，确保新上市公司进入第一轮初筛。

```bash
python3 scripts/fetch_a_share_universe.py \
  --output data/raw/a_share_securities.csv
```

执行要求：

1. 保留 `source_provider`、`source_dataset`、`source_retrieved_at_utc`、`source_url` 等来源字段。
2. 对比上一版 `data/raw/a_share_securities.csv`，识别本季度新增上市公司。
3. 新上市公司进入 `attention_triage`，但上市时间过短、披露材料不足时，不直接归为垃圾公司，应先归为 `boundary_pending`。
4. 更新证券名单这一动作不构成投资结论，但后续由此产生的新上市公司初筛结论必须写入 `data/processed/a_share_workflow_decision_log.csv`。

### 5.4 第一轮三类初筛

第一轮初筛只回答一个问题：这家公司以后是否值得继续消耗研究精力。它是轻量、全覆盖的第一层，对每一家A股给出 `worth_attention`、`boundary_pending` 或 `garbage` 之一。

#### 5.4.1 总体原则

1. **逐家独立判断**：每家公司单独分析，禁止用单一指标对多家公司批量归类。同业对比可作为判断的输入，但不能取代逐家判断。
2. **以竞争力为准**：基于公司实际竞争力判断，不以盈利水平或市值高低为主要依据。股价上涨、题材热度本身既不是入选证据，也不是排除证据。
3. **轻量初筛**：第一轮只判断业务模式、是否具备明显且可持久的竞争优势、是否存在明显负面；不要求逐家通读年报，明显垃圾可快速判定。通读年报、研报等深度分析留到对 `worth_attention` 做 L1-L5 分层和权威深度复核。
4. **入选从严**：`worth_attention` 的优势必须能通过资本复制测试——资金充足的新进入者难以快速复制并侵蚀其回报；“一定程度的护城河”须收紧为“持久、难以复制”。不设数量目标，数量由严格标准自然决定；存疑则不放入 `worth_attention`。
5. **排除分两种**：`garbage` 是永久排除，只给造假/治理灾难与结构性绝望行业；公司层面“当前无护城河但行业并非绝望”应放 `boundary_pending`（可逆），不要因一时平庸就永久排除。
6. **判不了就 boundary**：缺少可靠资料、无法判断实际水平、且无明显负面 → `boundary_pending`（含次新股披露不足）。

#### 5.4.2 三类定义

| 分类 | 定义 | 后续处理 |
| --- | --- | --- |
| `worth_attention` / 值得关注公司 | 具备能通过资本复制测试的持久优势：护城河、强市场地位、定价权与话语权；或竞争优势、政策优势、产业地位；或相对竞争对手有明确的差异化优势或整体优势。以竞争力为准，不看盈利/市值，从严。 | 进入质量分层（L1-L5）和后续估值排雷 |
| `boundary_pending` / 临界待定公司 | 两种情形之一：(a) 缺少可靠资料、暂时无法明确其实际水平、且无明显负面（含次新股披露不足）；(b) 可判断但当前缺乏持久优势，但所处行业并非结构性绝望，未来有发展或重获优势的可能。 | 不进入每日扫描；在出现可靠资料，或财报、公告、订单、客户验证、产品突破、重大转型、行业结构变化等硬触发时复核 |
| `garbage` / 垃圾公司 | 永久业务排除。只给：① 造假/治理灾难；② 结构性绝望行业。 | 永久排除，不再进入季度质量审查、质量分层、估值排雷和每日量价扫描，只维护证券基础信息 |

#### 5.4.3 进入 garbage 的条件

只在以下情形判为 `garbage`（永久排除）：

1. 造假/治理灾难：财务欺诈、重大虚假陈述、严重监管处罚、控股股东资金占用、严重审计异常，或管理层造假/严重不诚信。
2. 造假嫌疑：长期蹭热点、频繁炒作，公告与财报验证明显不匹配，存在造假或夸大宣传嫌疑。
3. 治理/财务长期不透明：现金流、应收、存货、负债、治理结构长期无法解释。
4. 结构性绝望行业：所处行业或细分本身低壁垒、不可能形成持久护城河——一旦证明能赚钱，资金充足的新竞争者可以轻易进入并迅速替代，行业内没有任何公司能建立持久优势。

注意：公司层面“当前同质化、暂无护城河”，若行业并非结构性绝望，应放 `boundary_pending`，不判 `garbage`；“被更强同行全面覆盖”属于后续分层与同业校准的考量，不在第一轮判 `garbage`。

`garbage` 一旦确认即永久排除，不因股价上涨、短期利润改善、媒体热度、主题炒作或低PE重新进入研究队列。只维护证券基础信息。

#### 5.4.4 输出字段

```text
security_code, security_name, attention_class, attention_reason,
capital_replicability_note, garbage_reason, garbage_subtype,
boundary_recheck_trigger, evidence_basis, reviewed_at_utc, workflow_version
```

`garbage_subtype` 取值：`governance_fraud` 或 `structural_industry`。

#### 5.4.5 行业校准锚

第一轮标准通过逐行业判例校准；每个行业的校准结论以可追溯方式记录在 `docs/peer-group-calibration/` 下对应文件，并在此沉淀可推广规则。

白酒行业校准（见 `docs/peer-group-calibration/a-share-baijiu.md` §1.10）得到的可推广规则：

1. **护城河需穿越周期验证**：`worth_attention` 要求护城河持久且已在完整下行周期中验证；护城河真实但穿越下行周期的能力尚未验证 → `boundary_pending`（可逆，验证后可升）。
2. **行业增长前景不是第一轮排除项**：行业长期增长走弱不在第一轮排除公司，只在 L1-L5 分层与估值中折价；第一轮以竞争力为准。
3. **传承品牌即资本难复制的护城河**：品牌 + 产区/原产地 + 老窖/不可替代实物资产 + 基酒/陈年库存 + 深耕渠道是典型持久护城河，此类行业并非结构性绝望。
4. **ST/困境的处理**：仅因连年亏损被 ST/困境、未坐实造假 → 不判 garbage，按竞争力判（通常 boundary）；`garbage(governance_fraud)` 需有坐实的造假/治理灾难，并核验公告。
5. **多元化拖累**：核心业务护城河真实、但被亏损非核心业务拖累、可比性差 → `boundary_pending`（剥离/重组后可逆），不直接 worth_attention 或 garbage。

面板行业校准（见 `docs/peer-group-calibration/a-share-display-devices.md` §1.5）补充：

6. **资本周期型制造业的护城河**（面板、半导体代工、光伏、LED、锂电等强周期、资本密集行业）：若龙头优势建立在**不可逆的结构性整合 + 客观巨大的资本/规模/技术壁垒**上（替代在客观上已很难），可判 `worth_attention`，即便穿越周期盈利尚未充分验证；周期性在 L1-L5 分层与估值折价。与规则 1 的区分：护城河**持久性本身存疑**（可能在需求下行中被侵蚀）时仍需穿越周期验证 → boundary；壁垒是客观结构性的则不需此前置证明。
7. **治理灾难的可逆性**：治理灾难发生在**前控股股东**、现已更换控制权并重整 → `boundary_pending`（控制权变更=复核触发），不判永久 garbage；`garbage(governance_fraud)` 留给未解决/仍在持续、或同一责任人仍在位的造假与治理灾难。


### 5.6 进入季度质量复核队列的条件

满足任一条件即进入队列：

1. 新上市公司。
2. 最新报告期晚于上次质量审查日期。
3. 原L1/L2公司。
4. 原L3公司出现业务、订单、利润率、技术路线、行业景气明显改善。
5. `boundary_pending` 公司出现重大业务变化、重大订单、关键产品突破、资产重组、客户验证、行业结构变化或连续财报改善。
6. 毛利率、净利率、经营现金流、负债率、研发投入、收入增速、利润增速出现重大变化。
7. 发生重大诉讼、监管处罚、审计异常、控股股东风险、财务疑点。

以下公司不进入队列：

1. `attention_class = garbage` 的公司。
2. `boundary_pending` 但没有硬触发的公司。
3. 只有股价上涨、传闻、小作文或主题热度，但没有公告、财报、客户验证、订单、权威研报或行业数据支持的公司。

### 5.7 质量分层标准

| 层级 | 定义 | 仓位资格 |
| --- | --- | --- |
| L1 | 顶级核心候选。行业或细分赛道核心龙头，具备极强护城河、技术壁垒、资源壁垒、品牌壁垒、平台能力或监管壁垒，资本充足的新进入者也难以短期复制和超越。 | 估值合适且右侧确认后，可承载15%-25%核心仓 |
| L2 | 准核心候选。护城河明确，但存在周期、政策、成长空间、业务结构、验证阶段或同业地位折扣。 | 估值合适且右侧确认后，可承载4%-10%准核心仓 |
| L3 | 战术观察。具备产业机会、反转机会、技术期权、局部壁垒或周期修复弹性，但护城河、现金流、市场地位或不可替代性不足以进入核心层。 | 只适合战术仓或事件动量仓 |
| L4 | 零仓观察。可理解、可偶尔复核，但护城河强度、催化清晰度、同业差异或证据质量不足以进入前三层。仍属值得关注。 | 默认不建仓，不进入每日扫描 |
| L5 | 移出值得关注、降级为临界待定（非永久排除）。低壁垒、易复制、被更强同行全面覆盖、现金流质量差，或主要靠题材/周期而无持久优势。判为L5表示不再值得长期关注，`attention_class` 由 `worth_attention` 降为 `boundary_pending`，等待未来硬触发再复核。 | 不建仓；永久排除的垃圾公司只在第一轮三类初筛产生，不用L5表达 |

### 5.8 质量审查执行规则

质量审查只判断业务质量，不考虑当前股价和估值。

输入材料：

1. 公司名称和股票代码。
2. 所属行业、同行组和主要竞争对手。
3. 最新年报、半年报、季报、业绩预告和业绩快报摘要。
4. 公司公告、交易所问询函、监管处罚、投资者关系材料。

## 6. 阶段二：核心质量公司估值排雷

### 6.1 执行范围

只对L1和L2公司执行估值分析。

### 6.2 输出分类

| 估值档位 | 含义 | 是否进入核心估值合格池 |
| --- | --- | --- |
| 低估 | 当前市值明显低于保守或正常估值，赔率充分 | 是 |
| 较低估 | 当前价格有明显吸引力，但存在周期、政策、业绩验证或行业折扣 | 是 |
| 中性 | 估值大致合理，未明显透支未来 | 是 |
| 可接受较高估 | 略贵，但bull case尚未完全定价，且后续仍可能上修 | 是 |
| 高估 | 已计入极端bull case，或需要过度乐观假设才能成立 | 否 |
| 无法估值 | 缺少足够数据或关键假设不可验证 | 否 |

### 6.3 估值原则

1. 估值只用于排除过度高估的核心质量公司。
2. 不因单纯PE高而排除处于产业爆发、研发投入期、盈利爬坡期的公司。
3. 不因单纯PE低而认定低估。
4. 对一次性亏损、一次性收益、研发投入、股权投资收益、公允价值变动、资产减值、补贴、会计处理变化进行归一化处理。
5. 对周期股使用中枢利润、资源NAV、商品价格中枢，不使用周期顶部单年利润。
6. 对产业链爆发股使用2-3年正常化利润和需求、瓶颈、利润三张票，不机械使用TTM PE。
7. 对早期高研发公司使用收入质量、毛利率成熟路径、研发效率、客户验证、产品壁垒和未来正常化利润，不机械使用当前亏损。

### 6.4 特殊估值失真处理

出现以下情形时，必须使用归一化估值：

1. 单季度或单年度存在一次性亏损。
2. 单季度或单年度存在一次性投资收益、公允价值收益或资产处置收益。
3. 公司处于研发投入期，当前利润被研发费用、产能爬坡、销售投入压低。
4. 公司处于行业周期底部，当前利润低于中枢利润。
5. 公司处于产业爆发前夜，当前利润尚未体现订单、产能、技术路线或客户验证。
6. 公司处于商业模式转换期，旧业务拖累利润，新业务尚未充分释放。

归一化估值字段：

```text
normalized_profit = 扣非净利润 - 一次性收益 + 一次性亏损调整 + 合理研发/产能爬坡归一化调整
normalized_margin = 成熟业务合理净利率
normalized_revenue = 可验证订单、产能、行业需求支持下的未来2-3年收入
normalized_market_cap = normalized_profit * 合理估值倍数
```

### 6.5 分策略估值方法

#### A：DCF / 现金流复利型

指标：

1. PE历史分位。
2. 自由现金流收益率。
3. 股息率和回购注销率。
4. ROE/ROIC稳定性。
5. 未来3年收入和利润增速。

预期年化收益：

```text
预期年化收益 = FCF收益率 + 保守利润增长率 + 股息/回购收益率 + 估值修复年化贡献
```

档位：

| 档位 | 标准 |
| --- | --- |
| 低估 | 预期年化收益 >= 12% |
| 较低估 | 8%-12% |
| 中性 | 6%-8% |
| 可接受较高估 | 4%-6%，且质量为L1或存在明确催化 |
| 高估 | <4%，或PE处于历史90%分位以上且增长放缓 |

#### B：烟蒂深度低估型

保守清算价值：

```text
保守清算价值 = 现金 + 0.8 * 应收账款 + 0.5 * 存货 + 0.3 * 固定资产 - 总负债
```

档位：

| 档位 | 标准 |
| --- | --- |
| 低估 | 市值 <= 保守清算价值 * 0.7，且有催化剂 |
| 较低估 | 市值 <= 保守清算价值 * 0.9，且资产真实 |
| 中性 | 市值接近保守清算价值，但催化剂不足 |
| 高估 | 账面资产真实性不足、负债压力大或无兑现路径 |

#### C：GARP成长型

指标：

1. 未来3年利润CAGR。
2. 归一化PEG。
3. 收入质量和市场空间。
4. 毛利率、净利率、经营现金流转化。
5. 增长是否可防守。

档位：

| 档位 | 标准 |
| --- | --- |
| 低估 | PEG <= 0.7，且增长可防守 |
| 较低估 | PEG 0.7-1.0 |
| 中性 | PEG 1.0-1.5 |
| 可接受较高估 | PEG 1.5-2.0，且盈利预测持续上修 |
| 高估 | PEG > 2.0，且无持续上修证据 |

#### D：产业链爆发 / 关键瓶颈型

必须同时评估：

1. 需求票：下游需求真实增长。
2. 瓶颈票：公司处于供给瓶颈或关键不可替代环节。
3. 利润票：需求和瓶颈能显著转化为上市公司收入和利润。

未来合理市值：

```text
未来合理市值 = 未来2-3年正常化净利润 * 合理PE
```

档位：

| 档位 | 标准 |
| --- | --- |
| 低估 | 当前市值 <= bull case合理市值 * 60% |
| 较低估 | 60%-80% |
| 中性 | 80%-100% |
| 可接受较高估 | 100%-120%，且盈利预测仍在上修 |
| 高估 | >120%，或bull case需要极端假设 |

#### E：落难白马型

指标：

1. 核心业务是否被结构性破坏。
2. 利润预期是否停止下修。
3. 现金流是否健康。
4. 管理层资本配置是否改善。
5. 估值是否低于正常化盈利水平。

档位：

| 档位 | 标准 |
| --- | --- |
| 低估 | 正常化利润对应PE显著低于历史中枢，且困境边际改善 |
| 较低估 | 正常化利润显示明显修复空间 |
| 中性 | 公司质量仍在，但触底信号不足 |
| 高估 | 仍处结构性恶化，或估值已提前反映修复 |

#### F：垄断资源型

指标：

1. 资源储量和NAV。
2. 成本曲线位置。
3. 商品价格中枢。
4. 穿越周期现金流。
5. 扩产能力和资本纪律。

档位：

| 档位 | 标准 |
| --- | --- |
| 低估 | 当前市值 <= 中枢价格NAV * 70%，或中枢FCF收益率 >= 10% |
| 较低估 | 当前市值 <= 中枢价格NAV * 85%，或中枢FCF收益率 7%-10% |
| 中性 | 接近中枢NAV或中枢利润合理估值 |
| 高估 | 必须依赖当前极端商品价格才能成立 |

#### G：股东回报型低估

指标：

1. PB/ROE匹配度。
2. 分红率。
3. 股息率。
4. 回购注销率。
5. 资产质量。

档位：

| 档位 | 标准 |
| --- | --- |
| 低估 | 股息率+回购率显著高于无风险利率，PB未透支ROE |
| 较低估 | 股东回报清晰，估值低于资产质量 |
| 中性 | 稳健但弹性有限 |
| 高估 | 低增长但估值已高于回报能力 |

### 6.6 估值审查执行规则

估值审查只对L1/L2核心质量公司执行，目标是排除过度高估，不是只寻找显著低估股票。估值必须先确定策略标签，再使用对应模型。

输入材料：

1. 公司名称、股票代码、质量分层和策略标签。
2. 最新价格、市值、PE、PB、股息率、回购率和历史估值区间。
3. 最新财报核心指标：收入、扣非净利润、经营现金流、毛利率、净利率、ROE/ROIC、资产负债率、应收、存货、资本开支。
4. 未来3年一致预期和权威研报核心假设。
5. 行业周期位置、商品价格中枢、产业链供需状态或渗透率阶段。
6. 一次性收益、一次性亏损、研发投入、产能爬坡、周期底部、商业模式转换等估值失真因素。
7. 当前市值与bull case合理市值、正常化利润估值或资源NAV的差距。

执行步骤：

1. 确认适用估值方法：DCF/现金流复利、烟蒂深度低估、GARP、产业链爆发、落难白马、垄断资源、股东回报型低估。
2. 判断当前PE/PB/PEG是否失真；如失真，使用归一化估值。
3. 写出估值模型关键输入，例如归一化利润、未来2-3年利润CAGR、合理PE、NAV、中枢FCF收益率、股息率或回购率。
4. 给出bull case核心假设，并判断当前价格是否已经反映bull case。
5. 分为：低估、较低估、中性、可接受较高估、高估、无法估值。
6. 判断是否进入核心估值合格池。
7. 给出等待右侧触发时需要观察的关键指标。
8. 将来源、结论、时间、输入文件、输出文件和简要判断理由写入结论日志。

估值模型和档位：

| 策略类型 | 模型 | 低估 | 较低估 | 中性 | 可接受较高估 | 高估 |

| GARP成长型 | 归一化PEG | PEG <=0.7且增长可防守 | 0.7-1.0 | 1.0-1.5 | 1.5-2.0且盈利预测持续上修 | >2.0且无持续上修证据 |
| 产业链爆发/关键瓶颈型 | 未来合理市值 = 未来2-3年正常化净利润 * 合理PE | 当前市值 <= bull case合理市值*60% | 60%-80% | 80%-100% | 100%-120%且盈利预测仍在上修 | >120%或bull case依赖极端假设 |

### 8.4 基础指标


MACD：


成交量指标：

```text
vol_ma5 = 5日平均成交量

```

### 8.5 有效放量定义

满足任一条件即可认定为有效放量：

1. 单日放量：
   ```text
   day_vol_ratio_20 >= 1.8
   ```

2. 连续放量：
   ```text
   vol_3d_ratio_20 >= 1.5
   或 vol_5d_ratio_20 >= 1.4
   ```

3. 启动前基准放量：
   ```text
   vol_5d_ratio_baseline >= 1.6
   且近5日至少3日成交量 > vol_baseline_60_21 * 1.2
   ```

4. 持续堆量：
   ```text
   vol_ma5 > vol_ma20 > vol_ma60
   且 vol_ma5 / vol_ma60 >= 1.5
   ```

5. 高分位放量：
   ```text
   vol_percentile_120 >= 80
   且近5日价格涨幅 > 3%
   ```

说明：连续多日放量会抬高20日均量，因此不能只用当日成交量/20日均量判断是否放量，必须同时使用启动前基准量、3日/5日合计量和成交量分位。

强月线确认：

```text
月收盘价 > 6月均线 > 10月均线 > 12月均线
```

### 8.7 右侧信号

**核心原则**：只要出现**有效放量上涨**（满足 §8.5 有效放量且当日收涨），无论是否突破前高或关键均线，即列入 `buy_candidate`（仍受 §8.8 过度延伸过滤与 §11 风险过滤）。前高突破、均线突破、相对强度等用于**提升优先级**，不是入选的必要条件；下列信号是放量上涨的细分形态，用于定级与排序。

#### 8.7.1 放量突破前高

前高周期：

```text
60日、120日、250日、500日、750日
```

触发：

```text
close > 对应周期前高 * 1.005
且满足有效放量
且收盘价位于当日振幅上半区
```

收盘位置：

```text
close_location = (close - low) / (high - low)
要求 close_location >= 0.6
```

#### 8.7.2 放量突破关键均线 / 趋势启动

前高突破不是买入候选的唯一触发条件。高质量股票在主升早段可能先表现为放量突破关键均线、均线系统转多头和持续承接，然后才突破长期前高。

触发：

```text
close > MA20
且 close > MA60
且满足有效放量
且收盘价位于当日振幅上半区
且满足日线多头或准多头
```

准多头：

```text
close > MA5 > MA10 > MA20
且 close > MA60
且 MA20斜率走平或向上
```

强确认：

```text
日线多头
或 最近3-5个交易日内放量冲高回落后重新收强
或 个股相对行业指数和沪深300明显走强
```

收盘位置：

```text
close_location >= 0.6
```

适用说明：

1. 如果已突破前高，优先按`8.7.1 放量突破前高`处理。
2. 如果未突破前高，但放量突破MA20/MA60并形成准多头或多头排列，可进入`buy_candidate`或`wait_confirmation`。
3. 如果当日放量但收盘位置低于0.6，先进入`wait_confirmation`，等待后续1-5个交易日重新收强。
4. 如果价格仍低于MA120或MA240，优先级低于突破长期前高的信号，但不自动排除。

#### 8.7.3 缩量涨停

涨停阈值：

```text
主板普通股票：涨幅 >= 9.5%
创业板/科创板：涨幅 >= 19%
北交所：涨幅 >= 29%
ST股票：涨幅 >= 4.5%
```

缩量条件：

```text
当日成交量 <= vol_ma20 * 1.2
或 在前一日已经放量后，当日成交量 <= 前一日成交量 * 0.85
```

#### 8.7.4 连续上涨

触发：

```text
最近5日内至少4日上涨
且5日累计涨幅在8%-25%之间
且日线多头
且没有单日极端放量长上影
```

#### 8.7.5 温和放量多头

触发：

```text
日线多头
且20日涨幅在3%-20%之间
且vol_5d_ratio_20在1.1-1.8之间
且MACD在最近3个交易日内金叉
```

#### 8.7.6 平台突破后二次确认

触发：

```text
前期已放量突破
随后3-10个交易日横盘或缩量回踩
回踩不跌破突破平台2%以上
再次放量创新高
```

#### 8.7.7 相对强度确认

触发：

```text
个股近20日收益率 > 沪深300近20日收益率 + 8pct
且个股近20日收益率 > 所属行业指数近20日收益率 + 3pct
```

### 8.8 过度延伸过滤

满足任一条件时，不直接列为买入候选，改为`wait_pullback`：

```text
close / MA20 - 1 > 25%
或 近5日涨幅 > 30%
或 近20日涨幅 > 60%
```

例外：

1. 同日或近期有明确财报、订单、客户认证、政策或产品催化。
2. 盈利预测出现重大上修。
3. 回踩平台不破后再次放量突破。

### 8.9 买入候选状态

| 状态 | 定义 |
| --- | --- |
| `buy_candidate` | 业务质量合格、估值未过度透支、出现有效放量上涨（无论是否破前高，含 §8.7 各右侧形态） |
| `wait_pullback` | 趋势强，但短期涨幅过大，等待回踩20日线或平台不破 |
| `wait_confirmation` | 单日异动明显，但收盘位置、持续承接或后续收强不足 |
| `wait_breakout` | 业务和估值合格，但尚未出现有效放量上涨 |
| `wait_repair` | 估值和业务合格，但跌破60日线或月线未修复 |
| `inactive_watch` | 仍在核心估值合格池中，但无买入信号 |
| `remove_watch` | 业务thesis或估值基础被证伪 |

### 8.10 买入候选优先级

| 优先级 | 条件 |
| --- | --- |
| S | L1，低估/较低估/中性，放量突破120日以上前高，日线多头，月线确认，相对强度确认 |
| A | L1/L2，低估/较低估/中性，出现放量突破前高、放量突破关键均线、温和放量多头或平台后二次确认 |
| B | L1/L2，可接受较高估，但盈利预测持续上修且出现强右侧信号 |
| C | L3战术观察股票出现强右侧信号，仅用于单独人工复核，不进入核心买入候选 |

说明：仅满足有效放量上涨、未突破前高或关键均线的，仍为 `buy_candidate`，但优先级低于上述突破类信号，按 §8.11 的中短期量价强势定级排序。

### 8.11 买入候选的强势跟踪

`buy_candidate` 一旦产生，必须每个交易日实时跟踪其股价与量价走势；强势程度**仅**由中期和短期量价走势判断，不引入基本面、前高位置等其它维度。

- **短期量价（约 5-10 日）**：close 相对 MA5/MA10，5 日涨幅，vol_5d_ratio_20，近 5-10 日相对沪深300与行业指数的强弱。
- **中期量价（约 20-60 日）**：close 相对 MA20/MA60，20-60 日涨幅，量能趋势（vol_ma5/vol_ma20/vol_ma60 排列），近 20-60 日相对强度。

| 强势 | 含义 | 处理 |
| --- | --- | --- |
| `strong` | 中短期量价同向走强、量价配合健康 | 维持买入候选，按 §8.10 优先级 |
| `neutral` | 中短期走平或量价背离 | 保持观察，等待再次放量上涨 |
| `weakening` | 中短期转弱、放量下跌或缩量阴跌 | 降级，必要时移出买入候选 |

强势跟踪只用量价，不构成买入命令；买入前仍需 §10 人工核对。

## 10. 买入前人工核对

任何`buy_candidate`在实际买入前必须完成以下核对：

1. 当日公告是否有真实催化。
2. 是否存在澄清公告、监管问询、减持、解禁、融资、财务异常。
3. 股价是否已经过度延伸。
4. 估值档位是否仍处于核心估值合格池。
5. 同行业是否存在更强替代标的。
6. 买入后如果失败，止损或降级条件是什么。

## 11. 固定风险过滤器

以下股票不进入每日买入候选：

1. 不在核心估值合格池中。
2. 估值档位为高估或无法估值。
3. 当前业务thesis被证伪。
4. 财务造假、审计异常、控股股东资金占用、重大治理风险。
5. 当日极端放量长上影且未收在高位。
6. 连续暴涨后远离20日线，且无明确业绩或订单上修。
7. 只因题材、小作文、传闻上涨，缺少公告、财报、订单、客户验证或权威研报支持。

## 12. 历史回放校准

### 12.1 校准目的

在脚本正式作为每日流程使用前，必须验证该流程能识别历史上的高质量右侧启动机会。

### 12.2 外部回放案例

具体回放案例不写入本流程文件。回放案例应作为外部验证集单独维护，避免把用于校准的历史样本固化为流程规则。

每个外部回放案例至少记录：

1. 回放股票代码。
2. 回放日期。
3. 当时已公开的公司资料、财报、公告、研报和行业数据。
4. 当时已发生的价格和成交量数据。
5. 该案例用于验证的流程环节：质量分层、估值排雷、量价触发或卖出规则。
6. 成功标准。

### 12.3 回放脚本接口

```bash
python3 scripts/backtest_signal_replay.py \
  --as-of YYYY-MM-DD \
  --symbols CODE1,CODE2 \
  --workflow-version a-share-selection-operation-v1 \
  --input data/processed/a_share_core_valuation_pool.csv \
  --output data/interim/replay_external_validation_YYYYMMDD.csv
```

说明：`backtest_signal_replay.py` 复用每日量价扫描规则，用于验证历史日期前的价格和成交量信号。完整历史回放必须提供当时已公开信息构造的 `--input` 核心估值合格池；如果使用当前核心估值合格池，则只能验证价格信号，不能证明当时已经满足质量和估值前置条件。

### 12.4 回放数据限制

回放只能使用回放日期之前已经公开的信息：

1. 当时已经披露的年报、季报、公告、业绩预告。
2. 当时已经发布的权威研报。
3. 当时已经发生的成交量、价格、指数数据。
4. 当时已经可见的行业政策和产业数据。

不得使用回放日期之后的财报、研报、订单、股价走势或事后总结。

### 12.5 回放失败后的固定调整

如果任一外部回放案例失败，按以下规则调整流程：

1. 对产业链爆发股和战略能力型公司，降低当前利润在估值中的权重。
2. 对处于研发、产能爬坡、商业模式切换、行业爆发前夜的公司，强制使用归一化利润和未来2-3年正常化利润。
3. 对强技术壁垒、客户验证、产能瓶颈、行业渗透率快速提升的公司，即使当前PE偏高，也允许进入核心估值合格池。
4. 对低利润或亏损公司，必须区分“真实能力尚未兑现”和“商业模式无效”。
5. 如果公司通过质量分层和估值排雷，但量价未触发，则保持核心估值合格池，不强行买入。

## 13. 固定仓位动作

每日量价扫描只输出操作偏向：

| 操作偏向 | 执行含义 |
| --- | --- |
| 可建目标仓位1/3 | 允许建立计划仓位的第一笔 |
| 等回踩 | 等待回踩20日线、60日线或突破平台不破 |
| 等确认 | 等待3-5个交易日持续承接 |
| 仅观察 | 不买入，只继续追踪 |
| 放弃 | 触发信号无效或估值/业务不合格 |

加仓必须满足：

1. 买入后趋势继续成立。
2. 回踩不破平台或均线。
3. 财报、订单、利润率、行业数据或盈利预测至少一项继续验证。
4. 当前估值仍未进入高估档。

退出必须满足：

1. 右侧失败，跌回突破平台。
2. 跌破60日线后3-5日不能收回。
3. 高位放量滞涨后连续缩量阴跌。
4. 业务thesis、估值假设或催化剂被证伪。
5. 出现重大财务或治理风险。

## 阶段五：每日持仓监控与卖出扫描

阶段四产出买入候选；阶段五对照当前真实持仓，每个交易日收盘后执行一次，把“宽出”操作化。卖出扫描不是卖出命令，执行卖出前仍需人工核对。

### 执行频率

每个A股交易日收盘后，与阶段四每日量价扫描同时执行。

### 输入

1. `data/processed/a_share_holdings.csv`：当前真实持仓，手工维护。
2. 前复权日线、周线、月线OHLCV。
3. 行业指数与沪深300指数。
4. `data/processed/a_share_core_valuation_pool.csv`：用于刷新持仓的最新估值档位。
5. `data/processed/a_share_workflow_decision_log.csv`：用于回看持仓的策略标签、bear case与退出条件。
6. 当日公告与重大新闻摘要。

### 持仓清单字段

`a_share_holdings.csv` 至少包含：

```text
security_code, security_name, strategy_tag, quality_tier, valuation_tier,
entry_date, build_amount_cny, initial_shares, cost_basis, current_shares,
stop_loss_price, stop_loss_last_raised_date, cumulative_trim_pct,
launch_platform_price, thesis_summary, bear_case, add_conditions,
exit_conditions, last_review_date, notes
```

字段说明：

- `build_amount_cny`：本次建仓投入金额，按“建仓金额规则”确定。
- `initial_shares`：建仓股数，作为盈利离场比例的基准；`current_shares` 为当前剩余股数。
- `cost_basis`：建仓均价。
- `stop_loss_price`：割肉价，买入时由用户给定，只能上调、不能下调。
- `cumulative_trim_pct`：已累计卖出占 `initial_shares` 的百分比。

### 建仓金额规则

1. 单次建仓金额 `build_amount` 初始为 25 万元；每只合适的买入标的用当前 `build_amount` 建仓，建仓后默认持有。
2. 当账户总资产达到 `build_amount * 20` 时，`build_amount` 翻倍（总资产 = 净资产 + 融资负债，即全部持仓市值含融资买入部分；原则上不持有现金）：

```text
总资产 < 500 万：build_amount = 25 万
总资产 >= 500 万：build_amount = 50 万
总资产 >= 1000 万：build_amount = 100 万
总资产 >= 2000 万：build_amount = 200 万
... 以此类推，每达到 build_amount * 20 翻倍一次
```

3. `build_amount` 只随总资产上台阶提高；已建仓位不因后续翻倍而追加。

### 割肉（止损）规则

1. 买入时必须在持仓清单写入 `stop_loss_price`（割肉价，由用户给定）。
2. 当日价 <= `stop_loss_price` 时，无条件清仓，不受 3 个月锁定限制。
3. 在触及割肉价之前，若持有时间 < 3 个月，禁止任何卖出；但出现“硬离场例外”时除外（见下）。
4. `stop_loss_price` 只能上调、不能下调；每次上调记录 `stop_loss_last_raised_date`。
5. 上调后的 `stop_loss_price` 不得高于当日 MA120，避免用过高的割肉价作为卖出借口。

**硬离场例外（可打破 3 个月锁定，立即清仓）**：出现以下任一情形时，由大模型判定后立即清仓，不受 3 个月锁定限制，也不必等待割肉价：

1. §16 一票否决：财务造假、控股股东资金占用、审计异常等确凿治理/财务问题。
2. 公司重大突发事件。
3. 季度报告披露后，利润严重不及预期。

### 盈利后离场规则

仅当持有时间 > 3 个月且当前浮盈为正时适用。基准为 `initial_shares`，按浮盈比例给出“允许累计卖出上限”；默认仍是持有——盈利本身不是卖出理由。

| 浮盈（相对建仓成本） | 累计允许卖出上限（占 `initial_shares`） |
| --- | ---: |
| >= +30% | 10% |
| >= +50% | 20% |
| >= +100% | 35% |
| >= +200% | 50% |

到 +200% 累计卖出 50% 时，已基本收回本金。`cumulative_trim_pct` 不得超过当前浮盈档对应的上限。这些是允许卖出的上限，不是强制卖出指令。

### 脚本接口

```bash
python3 scripts/scan_holdings_sell_signals.py \
  --as-of YYYY-MM-DD \
  --holdings data/processed/a_share_holdings.csv \
  --valuation-pool data/processed/a_share_core_valuation_pool.csv \
  --output-csv data/processed/daily_holdings_actions.csv \
  --output-md data/processed/daily_holdings_tracker.md \
  --log-file data/processed/a_share_workflow_decision_log.csv
```

脚本 `scripts/scan_holdings_sell_signals.py` 负责确定性部分：割肉触及、3 个月锁定、盈利离场资格、割肉价上调建议（不超 MA120）、建仓金额与单票权重。硬离场例外（一票否决/突发事件/季报利润严重不及预期）和是否上调割肉价由大模型判定。

### 脚本可判定的卖出与风险触发

按以下优先级处理每个持仓：

1. 硬离场例外（最高优先级）：大模型判定出现 §16 一票否决、公司重大突发事件、或季报利润严重不及预期 → `forced_exit`，立即清仓，打破 3 个月锁定。
2. 割肉清仓（无条件）：当日价 <= `stop_loss_price` → `stop_loss_sell`，全部清仓，不受 3 个月锁定限制。
3. 3 个月锁定：未触及上述例外或割肉价，且持有 < 3 个月 → 一律 `hold`，不输出任何卖出。
4. 盈利离场资格：持有 > 3 个月且浮盈为正 → 按“盈利后离场规则”计算当前允许累计卖出上限；若 `cumulative_trim_pct` 低于该上限 → `trim_eligible`，给出本次最多可卖比例（默认仍可选择持有）。
5. 割肉价上调建议：若现价显著高于 `stop_loss_price`，且当日 MA120 高于现 `stop_loss_price` → `raise_stop_suggested`，建议上调到不超过 MA120。

风险预警（仅提示，不直接触发卖出）：

1. 单票 > 30%、同一产业链 > 40%、同一大行业 > 60%、战术持仓合计 > 15%。
2. 账户自峰值回撤 -8% / -12% / -20% → 去杠杆 / 黄色警告 / 红色警告。
3. 跌破突破平台、跌破 MA60 且 3-5 日未收回、高位放量滞涨后缩量阴跌等技术走弱信号 → 作为上调割肉价的依据，不直接卖出。

### 大模型判定的卖出与风险信号

硬离场例外（判定成立即 `forced_exit`，立即清仓，打破 3 个月锁定）：

1. §16 一票否决：财务造假、控股股东资金占用、审计异常等确凿治理/财务问题。
2. 公司重大突发事件。
3. 季度报告披露后，利润严重不及预期。

其余信号仅用于上调割肉价与风险预警，不直接触发卖出（常规卖出只由割肉价和盈利离场上限决定）：

1. thesis 走弱：订单未兑现、客户验证失败、毛利未改善、需求证伪、竞争加剧、技术路线或政策逻辑变化、管理层口径与披露数据冲突。
2. 估值过热：当前价已反映 bull case、未来 2-3 年利润已被定价、股价远快于盈利上修。
3. 催化完成、机会成本变化等，作为是否上调割肉价或调整观察的参考。

### 持仓动作状态

| 状态 | 含义 |
| --- | --- |
| `hold` | 维持持有（默认，含 3 个月锁定期内） |
| `add_eligible` | 满足加仓条件，可按计划加仓 |
| `trim_eligible` | 持有 > 3 个月且浮盈达档，可在允许上限内部分了结 |
| `stop_loss_sell` | 触及割肉价，无条件清仓 |
| `forced_exit` | 一票否决/突发事件/利润严重不及预期，立即清仓，打破 3 个月锁定 |
| `raise_stop_suggested` | 建议上调割肉价（不超过 MA120） |
| `risk_alert` | 出现暴露超限、回撤预算或治理/财务风险，需复核 |

### 卖出前人工核对

任何 `stop_loss_sell`、`forced_exit`、`trim_eligible` 在实际执行前必须核对：

1. 当日是否有澄清公告或被误读的利空。
2. 是否处于不可交易状态（停牌、一字跌停）。
3. 触发是否为单日噪声，是否需等待1-3个交易日确认。
4. 减仓比例是否符合盈利后离场规则的允许上限。
5. 卖出后若反向走强的再买回条件。

### 与买入侧的衔接

1. 阶段四产生 `buy_candidate` 并实际建仓后，必须把该持仓写入 `a_share_holdings.csv`，记录策略标签、`build_amount_cny`、`initial_shares`、`cost_basis`、`stop_loss_price`、突破平台价、bear case与退出条件。
2. `a_share_holdings.csv` 与买入候选相互独立：买入候选是池内信号，持仓清单是已建仓事实。
3. 每个持仓动作结论写入决策日志，`workflow_stage = holdings_sell_scan` 或 `manual_sell_check`。

## 14. 执行方式

本文件是A股选股、估值、量价扫描的主执行规范。后续可以用类似以下指令执行固定流程：

```text
请基于 docs/a-share-selection-operation-workflow.md 执行今天的A股每日量价扫描，并将结果写入固定输出文件和结论日志。
```

执行分工：

1. 能由脚本稳定完成的部分必须用脚本执行，例如证券名单更新、队列生成、核心估值合格池物化、每日量价指标扫描、每日持仓技术与风险触发扫描和历史回放。
2. 需要判断业务质量、护城河、估值模型、研报假设、公告真实性和用户defence的部分，由大模型按本文件中的执行规则和输出字段执行。
3. 大模型执行时不得临时改标准；如发现标准不合理，先修改本文件，再按新版本重新执行。
4. 每个产生结论的脚本输出和每个大模型批次结论都必须写入 `data/processed/a_share_workflow_decision_log.csv`；队列生成脚本只做调度，不单独写结论日志。
5. 每日量价扫描不是买入命令。买入前仍需完成第10节人工核对。
6. 每日持仓卖出扫描不是卖出命令。卖出只由割肉价和盈利后离场上限决定；卖出前仍需完成阶段五的人工核对。
