两种策略对比分析

一、定位与核心目标

维度 A股选股-估值-量价操作流程（下称"工作流"） 股票通用分析框架 v2.1（下称"框架v2.1"）
定位 全市场系统性选股+持仓管理流水线 单只股票深度分析决策工具
核心问题 “全市场5000+只股票中，今天该关注哪些、该买什么、该卖什么” “给定一只股票，值不值得买、怎么买、怎么卖”
适用场景 有精力做全市场扫描的专职/半专职投资者 收到线索后对个股做深度研判的投资者
时间跨度 跨季度到跨年（质量分层季度更新+每日量价扫描） 中线1-12个月
产出物 10+种CSV/MD文件、决策日志、每日买入候选/持仓动作 一张可执行的决策卡

本质差异：工作流解决的是"选股漏斗"问题——从全市场逐级过滤到买入候选；框架v2.1解决的是"个股裁决"问题——对一只股票做五步审判后输出决策卡。两者是互补关系而非替代关系。

二、流程架构对比

维度 工作流（5阶段） 框架v2.1（5步+风险矩阵）
阶段1 季度全市场质量审查：三类初筛（worth_attention/boundary/garbage）→ L1-L5分层 业务拆解+行业周期定位：六问+业务结构表+周期定位+催化剂地图
阶段2 核心质量公司估值排雷：7种策略（A-G）分档估值 护城河与竞争格局：五力自检+量化指标+竞争对标+侵蚀风险
阶段3 财报披露后滚动更新 财务体检+收入质量：四维体检+增长质量+收入拆解+资本配置
阶段4 每日量价扫描：7种右侧信号+信号分级+买入候选输出 估值定价+隐含预期：三情景估值+反向DCF+适用性检验
阶段5 每日持仓监控与卖出扫描：割肉+趋势保护+盈利离场 交易计划与风控：形态分析+分仓+三维止损+信心加权仓位

关键区别：

  • 工作流的阶段1-3是全市场批量处理，阶段4-5才是每日执行；
  • 框架v2.1的五个步骤是针对单只股票的线性流程，一次性走完产出决策卡；
  • 工作流的"质量筛选"和"估值排雷"是分离的两个阶段，框架v2.1把它们融入"护城河评估"和"财务体检"中。

三、优势对比

工作流的优势（框架v2.1不具备的）

1. 全市场覆盖能力

  • 工作流从交易所全A股名单出发，逐家初筛，不遗漏任何公司
  • 框架v2.1只能分析"你已经知道的股票"，没有选股发现机制

2. 系统化数据基础设施

  • 10+种固定产物文件（证券名单、初筛结果、质量分层、估值池、买入候选、持仓动作、决策日志等）
  • 所有结论写入决策日志，可溯源、可复盘、可纠错
  • 框架v2.1的决策卡是一次性的，没有历史对比和滚动更新机制

3. 脚本化执行

  • 证券名单更新、队列生成、每日量价扫描、持仓监控、历史回放都有脚本接口
  • 确定性部分由脚本完成，判断部分由大模型/人工完成
  • 框架v2.1完全依赖人工分析，无自动化

4. 极其精细的量价信号体系

  • 7种独立右侧信号（放量突破前高/放量突破均线/缩量涨停/连续上涨/温和放量多头/平台二次确认/相对强度）
  • 每种信号有精确的数学公式（如day_vol_ratio_20 ≥ 1.8、close_location ≥ 0.6）
  • 信号分级（强/中/弱）×优先级（S/A/B/C）的二维矩阵
  • 框架v2.1的形态分析（W底/头肩底/旗形）偏定性，缺乏量化公式

5. 精细的持仓管理纪律

  • 3个月锁定机制（防止频繁交易）
  • 割肉价只能上调不能下调（防止心理锚定下移）
  • 三档趋势保护线（MA60战术/MA150爆发/MA250复利）按策略类型差异化
  • 盈利后离场规则（+30%可卖10%、+50%可卖20%…梯度释放）
  • 建仓金额自动翻倍机制（随总资产增长扩大单笔规模）
  • 框架v2.1的止损止盈是原则性描述，没有这种粒度的制度化约束

6. 质量分层的金字塔结构

  • L1-L5五档分层，每档有明确的仓位资格（L1可15-25%、L2可4-10%…）
  • 校准细则防止"中间桶"泛滥（L3必须有明确战术理由，L4是保守默认）
  • 框架v2.1只有"通过/警示/否决"三色门，没有中间档的精细区分

7. 历史回放校准机制

  • 可以用历史数据验证流程是否能识别过去的优质右侧启动点
  • 框架v2.1没有回测/验证机制

框架v2.1的优势（工作流不具备的）

1. 收入质量拆解（经常性vs一次性）

  • 明确区分产品销售、BD授权首付款、里程碑款三种收入类型
  • 提供"经常性PE"计算方法
  • 工作流虽然有"归一化估值"概念（§6.4），但没有将收入类型三分类的制度化拆解

2. 反向DCF/隐含预期拆解

  • 用当前市值反推市场隐含增速，再与自己的判断对比发现预期差
  • 工作流的估值只做正向估值（“我认为值多少”），没有反向拆解（“市场认为值多少”）

3. 估值方法适用性检验

  • 明确列出每种估值方法的适用场景和失效场景
  • PE不适用于周期股/Biotech/含一次性收入的公司——这条规则在工作流中只是隐含的
  • 工作流有7种估值策略但没有显式做"这个方法对这家公司是否有效"的前置检验

4. 行业周期定位模块

  • 导入/成长/成熟/衰退 + v2.1新增的"产能/技术周期"（CapEx拐点+新技术标准落地）
  • 工作流在估值阶段会考虑行业周期位置，但没有独立的周期定位模块

5. 催化剂地图

  • 5类催化剂（业绩兑现/产品里程碑/政策事件/BD合作/竞争格局变化）×预计时点×影响程度×是否已定价
  • 工作流在买入前人工核对中提到"当日催化"，但没有系统化的催化剂地图

6. 风险因子矩阵（前置风险管理）

  • 政策/竞争/财务/治理/宏观5类风险 × 概率×影响 → 预设应对
  • v2.1新增"估值锚失效"风险（市场审美切换导致估值体系变化）
  • 工作流的风险管理主要是事后的（止损/趋势保护/硬离场），前置的"什么可能出错"分析散落在各阶段，没有独立矩阵

7. 技术形态分析

  • W底/头肩底/圆弧底/上升旗形/M头等具体形态与买卖点对应
  • 形态破位止损（假突破3日内认错、跌破颈线果断落袋）
  • 顶背离止盈信号
  • 工作流的量价信号是纯统计指标（放量比例/均线位置），不涉及图表形态

8. 信心加权仓位

  • 五步决策门通过情况 → 信心系数（0.5/1.0/1.2）→ 调整仓位
  • 工作流的仓位由L1-L5分层决定，但没有"分析通过度"影响仓位的机制

9. 决策卡模板

  • 一张卡浓缩五步结论：标的/逻辑/五步评估/催化剂/关键价位/分仓路径/止损止盈/失效条件
  • 工作流的输出是CSV/MD文件，没有这种人可读的"一张卡执行"格式

10. 通用性

  • 适用于A股/港股/美股，不限市场
  • 工作流专为A股设计，均线参数（MA5-MA250）、涨跌停规则、除权处理都绑定了A股特征

四、劣势对比

工作流的劣势

劣势 说明
只适用A股 均线参数、涨跌停阈值、除权规则、沪深300市场状态判定都绑定A股
缺乏形态分析 纯统计量价信号，不识别W底/头肩底等图表形态，对主观交易者不友好
无反向DCF 只做正向估值，不知道"市场当前价格隐含了什么预期"
无催化剂地图 催化剂散落在各处，没有系统化的"持有期内跟踪什么"机制
无风险因子矩阵 前置风险管理没有独立模块化
执行门槛极高 需要维护10+种CSV文件、运行多个脚本、季度全市场扫描——只适合有大量时间的专职投资者
质量分层主观性强 L1-L5分层虽然规则详细，但最终仍依赖大模型/人工判断，14条边缘判例说明争议频繁
估值方法虽多但偏机械 7种策略各有阈值表格，但缺乏"这个方法对这家公司是否有效"的前置检验

框架v2.1的劣势

劣势 说明
无选股发现机制 只能分析"你已经知道的股票"，无法从全市场筛选标的
无数据基础设施 没有滚动更新、决策日志、历史对比——每次分析都是一次性的
无持仓管理制度 没有锁定期、割肉价只能上调、盈利离场梯度等制度化约束
量价信号偏定性 形态分析依赖人工识别，缺乏工作流那种精确的数学公式
无回测验证 不能用历史数据验证框架有效性
无市场状态判定 不判断大盘强势/弱势/震荡，操作偏向不随市场环境调整
仓位管理粒度粗 信心系数（0.5/1.0/1.2）只有三档，远不如工作流L1-L5×估值档位×信号分级的精细矩阵
不适合批量处理 每只股票都要走五步，无法高效覆盖全市场

五、互补性分析

两个策略高度互补，可以组合使用：

环节 用哪个 理由
选股发现 工作流 全市场扫描→三类初筛→L1-L5分层，框架v2.1无此能力
个股深度分析 框架v2.1 收入质量拆解、反向DCF、催化剂地图、风险因子矩阵比工作流更深
估值定价 两者结合 工作流的7种策略提供方法选择，框架v2.1的适用性检验+反向DCF提供交叉验证
买入时机 两者结合 工作流的量价信号提供精确的数学触发，框架v2.1的形态分析提供图形直觉
持仓管理 工作流 3个月锁定、割肉价规则、趋势保护线、盈利离场梯度远比框架v2.1制度化
风险管理 两者结合 工作流的事后止损纪律+框架v2.1的事前风险因子矩阵=完整闭环
复盘改进 工作流 决策日志+历史回放提供系统性复盘能力

理想的组合用法：

  1. 用工作流做全市场扫描，产出L1/L2核心估值合格池
  2. 对池中出现买入信号的股票，用框架v2.1做深度分析（收入质量拆解、反向DCF、催化剂地图、风险因子矩阵）
  3. 框架v2.1的决策卡通过后，按工作流的建仓金额规则和分仓路径执行
  4. 持仓期间按工作流的持仓管理制度执行（锁定、趋势保护、盈利离场）
  5. 每日用工作流的量价扫描跟踪买入候选和持仓，用框架v2.1的催化剂地图验证逻辑是否还在
