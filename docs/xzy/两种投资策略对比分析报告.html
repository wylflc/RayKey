两种投资策略详细对比分析报告

A股选股-估值-量价操作流程 (v15) vs 股票通用分析框架 v2.1 · 全维度深度解析

TL;DR · 核心结论

两份文档描述的是两种定位完全不同的投资体系，而非同一类方法的两个版本。A股选股-估值-量价操作流程（以下简称"操作流程"）是一套系统化、可重复执行的量化+基本面混合体系，目标是全A股约5000家公司的批量筛选、长期跟踪与纪律化买卖；股票通用分析框架 v2.1（以下简称"分析框架"）是一套针对单只股票的五步决策方法论，目标是从研究到交易把一只股票彻底分析清楚并输出可执行决策卡。两者并非互斥——操作流程负责"全市场筛选+持仓管理"，分析框架负责"个股深度研判+单笔交易计划"，在实战中可以互补使用。

1设计哲学与定位对比

A股选股-估值-量价操作流程 (v15)

定位

一套可工程化执行的操作规范，强调"固定产物、固定脚本、固定规则"。

哲学

  • 业务质量决定是否值得长期跟踪——先做全市场质量筛选
  • 估值用于排除过度透支——而非只寻找低估
  • 量价信号只在高质量+估值合格池内触发——三层漏斗递进
  • 每日输出买入候选而非买入指令——人工核对后再执行
  • 每一次结论必须写入决策日志——可溯源、可复核、可纠错

本质

把投资变成流水线作业：确定性部分交给脚本，判断部分交给大模型，人只做最终核对。追求的是纪律性和可重复性。

股票通用分析框架 v2.1

定位

一套可套用于任意个股的完整决策方法论，强调"五步流水线+三色决策门"。

哲学

  • 先筛掉不该碰的公司——前三步是排除法
  • 再筛掉太贵的公司——第四步是估值把关
  • 最后把剩下的变成有纪律的单子——第五步是执行
  • 前一步否决就不必走下一步——节省精力、降低风险
  • 没有止损位的计划不是计划——强制风控

本质

把投资分析变成结构化思考工具：每一步都有明确的检查清单和决策门，最终产出一张"拿到就能下单"的决策卡。追求的是分析深度和决策完整性。

关键差异：操作流程是"先有池子，再等信号"的自上而下体系——它先建立一个高质量的核心估值合格池，然后每日用量价信号在池中触发买入候选；分析框架是"先有标的，再做尽调"的自下而上体系——它从一只具体的股票出发，走完五步后决定买不买、买多少。两者代表了两种完全不同的投资工作方式。

3质量评估体系对比

3.1 操作流程的质量评估

操作流程采用三层漏斗式质量筛选，极其精细：

全A股名单
→ 第一轮三类初筛：worth_attention / boundary_pending / garbage
→ (仅worth_attention) L1-L5质量分层
→ 核心质量公司 = L1 + L2
→ 核心估值合格池 = L1/L2中估值未透支的股票

其质量评估有三大特色：

  • 资本复制测试：worth_attention的护城河必须能通过"资金充足的新进入者能否快速复制并侵蚀其回报"的测试，这是核心判定标准。
  • 行业校准锚：沉淀了12条可推广规则和14组对照判例（如"长电worth、华能水电boundary"），用同行业判例校准标准，避免主观随意。
  • 防过度纳入清单：7条否决项明确列出"研究价值≠值得观察""强位置≠超额回报护城河"等，从严控制worth_attention数量。

3.2 分析框架的质量评估

分析框架采用护城河五力 + 量化验证 + 竞争对标的组合方式：

  • 护城河五力自检：品牌/无形资产、成本优势、网络效应、转换成本、规模/牌照/资源，每条分强/中/弱。
  • 硬指标门槛：ROIC连续5年>15%、ROE连续5年>15%、自由现金流持续为正、市占率上升/稳定。
  • 竞争对手量化对标：选2-3家可比公司拉5年数据对比ROIC、毛利率、市占率、收入CAGR、估值。
  • 护城河侵蚀风险：前瞻性评估3年内技术替代、政策变化、携资跳入、人才流失的概率和影响。

3.3 质量评估对比小结

维度操作流程 (v15)分析框架 v2.1
评估粒度极细——L1-L5五档分层，每档有明确的仓位资格(15-25%/4-10%/战术仓/零仓/移出)中等——通过/警示/否决三档，依赖分析者主观判断强度
同业校准系统化——12条规则+14组判例+行业校准锚文件建议性——选2-3家可比公司对标，无固定判例库
否决机制双重否决——防过度纳入清单(7条)+一票否决(造假/治理)三色门——护城河否决=仅可短线交易
可复现性高——逐家独立判断+决策日志，可复核纠错低——依赖分析者经验，不同人结论可能不同
前瞻性较弱——以当前竞争力为准，穿越周期验证为辅较强——专设"护城河侵蚀风险"前瞻评估

4估值方法对比

4.1 操作流程的估值体系

操作流程定义了7种分策略估值方法，每种都有明确的档位标准：

策略类型核心模型
ADCF/现金流复利型预期年化收益 = FCF收益率 + 保守利润增长率 + 股息/回购收益率 + 估值修复
B烟蒂深度低估型保守清算价值 = 现金 + 0.8×应收 + 0.5×存货 + 0.3×固定资产 - 总负债
CGARP成长型归一化PEG
D产业链爆发/关键瓶颈型未来合理市值 = 未来2-3年正常化净利润 × 合理PE
E落难白马型正常化利润和修复后合理PE
F垄断资源型资源NAV、中枢商品价格、中枢FCF收益率、成本曲线
G股东回报型低估PB/ROE、股息率、回购注销率、资产质量

其估值体系的核心原则是"估值只用于排除过度高估，不是只寻找显著低估"，并设有完善的归一化估值机制处理一次性亏损/收益、研发投入期、周期底部等失真场景。

4.2 分析框架的估值体系

分析框架的估值体系更注重方法选择和预期拆解：

  • 公司类型→方法映射：高成长(PEG/PS)、稳健成长(PE/PEG)、周期(PB/中枢PE)、价值成熟(PE/PB/股息率)、平台金融(SOTP/DCF)、Biotech(rNPV)、周期拐点(正常化PE)。
  • 估值三重判断：历史纵向分位 + 行业横向对比 + 预期拆解（说清"已定价什么"）。
  • 三情景估值：Bear/Base/Bull分别假设，按25%/50%/25%加权得到合理价值中枢。
  • 反向DCF：用当前市值反推隐含增速，对比自己判断发现预期差——这是分析框架的独特优势。
  • 估值方法适用性检验：明确每种方法的失效场景和替代方法，防止"用PE估周期股"等典型错误。

4.3 估值方法对比小结

维度操作流程 (v15)分析框架 v2.1
估值策略数7种(A-G)，每种有量化档位标准按公司类型映射，无固定档位表
目标排除高估（低估/较低估/中性/可接受较高估 → 入池）寻找性价比（>20%上行空间+分位<50% → 通过）
归一化处理完善——6种失真场景+归一化公式有——收入质量拆解(经常性vs一次性)
反向DCF无（以正向估值为主）有——核心特色，发现预期差
适用性检验隐含在策略选择中显式——独立子模块，明确失效场景
可执行性高——档位标准量化、脚本可物化中——需分析者自行填表判断
估值维度结论：操作流程的估值体系更系统化、可量化，适合批量处理；分析框架的估值体系更有深度、防误导，特别是反向DCF和估值适用性检验能有效避免"用错方法得出错误结论"。两者在归一化处理上都重视一次性收入的剔除，但分析框架用三生国健案例做了更生动的说明。

5买入时机与信号系统对比

5.1 操作流程的量价信号系统

操作流程拥有极其精细的右侧量价信号系统，这是其最核心的特色之一：

  • 有效放量定义：5种判定方式（单日放量/连续放量/启动前基准放量/持续堆量/高分位放量），避免单一指标误判。
  • 7种右侧触发形态：放量突破前高(8.7.1)、放量突破关键均线(8.7.2)、缩量涨停(8.7.3)、连续上涨(8.7.4)、温和放量多头(8.7.5)、平台突破二次确认(8.7.6)、相对强度确认(8.7.7)。
  • 信号分级：强/中/弱三档，按"放量×短期多头×突破确认"三层递进。
  • 买入候选优先级：S/A/B/C四档，按"质量×估值"维度划分，与信号分级正交。
  • 过度延伸过滤：close/MA20-1>25%或近5日涨幅>30%时改为wait_pullback，防止追高。
  • 市场状态判定：用沪深300 vs MA250判定强势/弱势/震荡，弱势市场各档偏向下调一档。
  • 均线体系：统一7条均线(MA5/10/20/60/120/150/250)，每条用途唯一且必要。

5.2 分析框架的交易时机系统

分析框架的买入时机判断更偏定性、重形态：

  • 交易位置判断：低位启动/趋势中继/高位加速/高位震荡/下跌反抽五类，每类有特征+形态+量能+策略倾向。
  • 形态分析：筑底形态(W底/头肩底/圆弧底)、整理形态(上升旗形/箱体/上升三角形)、顶部形态(M头/头肩顶)。
  • 资金流向确认(v2.1新增)：ETF净流入、龙虎榜机构买盘、北向/南向资金净流入作为右侧确认。
  • 分仓路径：试错仓(10-20%) → 确认仓(+20-30%) → 加仓仓(+20-30%) → 上限(≤70%)。
  • 核心原则："形态只有配合量能才有意义——量在价先"，"形态只是辅助工具，不能取代基本面"。

5.3 买入时机对比小结

维度操作流程 (v15)分析框架 v2.1
信号定义极其量化——5种放量+7种形态+3档分级，脚本可自动判定偏定性——形态+量能+资金流向，需人工综合判断
触发前提必须在核心估值合格池内（质量+估值双重前置）必须通过前四步决策门（业务+护城河+财务+估值）
防追高机制过度延伸过滤——量化阈值(close/MA20-1>25%等)交易位置判断——高位加速/高位震荡时"不追，等回踩"
市场环境有——沪深300 vs MA250判定市场状态，自动调整操作偏向无独立机制——风险矩阵中有"大盘系统性下跌"但不自动调整
分仓逻辑建仓金额规则(25万起)+L1/L2仓位资格(15-25%/4-10%)试错→确认→加仓三段式+信心加权系数(0.5/1.0/1.2)
买入时机结论：操作流程的量价系统远比分析框架精细和可执行，特别是7种右侧触发形态和信号分级机制，是经过大量实战打磨的量化体系。分析框架的形态分析更像是"辅助参考"，而操作流程的量价系统是"核心引擎"。如果只看买入时机的精细度，操作流程完胜。

6风控与卖出体系对比

6.1 操作流程的风控卖出体系

操作流程拥有极其完整的卖出扫描体系，这是"宽出"理念的工程化实现：

  • 割肉价（无条件清仓）：买入时用户给定，只能上调不能下调，上调不超MA120。触及即清仓，不受3个月锁定限制。
  • 3个月锁定期：买入后3个月内禁止任何卖出（硬离场例外除外），强制持有，避免情绪化操作。
  • 硬离场例外：一票否决(造假/治理)/重大突发事件/季报利润严重不及预期，可打破3个月锁定立即清仓。
  • 趋势保护线（分档软性减仓）：
    持仓类型保护线跌破定义触发动作
    daily（战术仓）MA60或平台连续3日收盘低于线且距线>3%退出复核
    weekly（B/C/D/E）MA150（约30周）连续5日收盘低于线且距线>3%减半复核
    monthly（A/F/G）MA250（年线）连续10日收盘低于线且距线>5%降档减仓复核
  • 盈利离场规则：浮盈>=30%可卖10%、>=50%可卖20%、>=100%可卖35%、>=200%可卖50%，是允许上限非强制。
  • 风险预警：单票>30%、同产业链>40%、同大行业>60%、战术仓>15%；账户回撤-8%/-12%/-20%三级响应。
  • 除权除息处理：机械调整价格与股数字段，避免除权日误触发。

6.2 分析框架的风控卖出体系

分析框架的风控更注重前置风险管理和多维止损：

  • 三维止损框架：
    • 价格止损+形态破位：跌破前低/20日线/关键位，幅度止损-5%至-8%
    • 逻辑止损：核心催化失效、利好落空、板块龙头跌停、业绩证伪
    • 时间止损：短线5-10个交易日未验证，中线1-2个月未兑现逻辑
  • 分批止盈：到Base目标价卖1/3、到Bull目标价再卖1/3、剩1/3做第二段；情绪止盈(换手>15%+放量滞涨)、形态止盈(顶背离/M头颈线)。
  • 风险因子矩阵（独立模块）：政策/竞争/财务/治理/宏观五大类风险，每个评估概率×影响→预设应对方案，在买入前完成。
  • 失效条件：催化证伪、板块转弱、放量破位、次日反馈差、环境不支持、形态破坏——任一触发即认错。
  • 止损纪律红线："触发后禁止重新评估持有理由——评估应在触发前完成"。

6.3 风控卖出对比小结

维度操作流程 (v15)分析框架 v2.1
卖出体系完整度极高——割肉价+锁定+趋势保护+盈利离场+硬离场，五层防线高——三维止损+分批止盈+风险矩阵+失效条件

风控结论：操作流程的卖出体系在执行纪律和精细度上远超分析框架——特别是割肉价+3个月锁定+分档趋势保护线的组合，能有效防止"触发止损后又找理由扛着"。但分析框架的风险因子矩阵是独特优势——它把风控从"被动触发"升级为"主动前置"，在买入前就预设应对方案，这是操作流程所缺乏的。理想状态是：用分析框架做买入前风险梳理，用操作流程做持仓期间纪律执行。

7仓位管理对比

操作流程的仓位管理

建仓金额规则

总资产 < 500万：build_amount = 25万
总资产 >= 500万：build_amount = 50万
总资产 >= 1000万：build_amount = 100万
总资产 >= 2000万：build_amount = 200万
每达到 build_amount × 20 翻倍一次

仓位资格（按质量分层）

  • L1：估值合适+右侧确认 → 15%-25%核心仓
  • L2：估值合适+右侧确认 → 4%-10%准核心仓
  • L3：只适合战术仓或事件动量仓
  • L4：默认不建仓

加仓条件

趋势继续成立+回踩不破+财报/订单/利润率/盈利预测至少一项验证+估值未进高估档。

分析框架的仓位管理

仓位计算公式

单笔风险预算 = 账户总资金 × 单笔最大可承受亏损%(1-2%)
基础仓位 = 单笔风险预算 ÷ (买入价 - 止损价) ÷ 买入价
实际仓位 = 基础仓位 × 信心系数

信心系数：
四步全通过+形态共振 = 1.2
三步通过+一步警示 = 1.0
两步通过+两步警示 = 0.5
任一步否决 = 0

分仓路径

  • 试错仓：10-20%（首次信号）
  • 确认仓：+20-30%（走势符合预期）
  • 加仓仓：+20-30%（强势确认）
  • 上限：≤70%
维度操作流程 (v15)分析框架 v2.1
仓位计算逻辑按质量分层(L1/L2)给固定仓位资格区间按风险反推(单笔风险预算÷止损距离)×信心系数
大资金适配强——build_amount随总资产翻倍，天然适配大资金弱——按账户百分比计算，大资金需自行调整
止损与仓位关联弱关联——割肉价独立设定，仓位由分层决定强关联——"没有止损位就无法计算仓位"
信心权重无——质量分层本身已隐含信心有——信心系数(0.5/1.0/1.2)显式加权

9优势与劣势总结

9.1 操作流程 (v15) 的优势与劣势

核心优势
  • 全市场覆盖：5000+家A股全覆盖初筛，不遗漏任何潜在标的
  • 质量分层极精细：L1-L5五档+12条校准规则+14组判例，标准可追溯
  • 量价信号系统极强：7种右侧形态+5种放量定义+3档分级，实战打磨
  • 卖出纪律极严：割肉价+3个月锁定+分档趋势保护，防止情绪化操作
  • 高度自动化：脚本化执行+决策日志+跨轮次索引，可复现可追溯
  • 大资金适配：build_amount翻倍规则天然适配资金增长
  • 版本迭代成熟：v1到v15持续打磨，边缘判例丰富
主要劣势
  • 仅限A股：脚本、规则、均线体系均针对A股设计，无法用于港股/美股
  • 缺乏反向DCF：以正向估值为主，无隐含预期拆解机制
  • 缺乏前置风险矩阵：卖出扫描是每日被动执行，无买入前主动风险梳理
  • 护城河侵蚀评估较弱：以当前竞争力为准，前瞻性侵蚀评估不如分析框架
  • 形态分析较弱：量价信号偏纯技术指标，缺乏W底/M头等经典形态分析
  • 初始建设成本高：需完成全市场初筛+分层+估值才能运转，前期工作量大
  • 不适合快速决策：流程重，不适合"今天看到一只股票，明天就要决定"的场景

9.2 分析框架 v2.1 的优势与劣势

核心优势
  • 跨市场通用：A股/港股/美股均可套用
  • 分析深度强：五步流水线从业务到交易全覆盖，每步有检查清单
  • 反向DCF独特：隐含预期拆解能发现市场预期差，避免"我觉得值多少"的主观陷阱
  • 估值适用性检验：显式防止"用PE估周期股/Biotech"等典型错误
  • 风险因子矩阵：买入前主动梳理五类风险，预设应对方案，前置防御
  • 收入质量拆解：经常性vs一次性收入区分，三生国健案例生动
  • 形态分析丰富：W底/M头/旗形等经典形态+资金流向确认，买点具象化
  • 信心加权仓位：决策门通过情况与仓位显式关联，逻辑清晰
  • 上手快：拿到一只股票即可走五步，无需前期建池
主要劣势
  • 无全市场筛选能力：只能分析单只股票，无法主动发现标的
  • 无自动化：纯人工填表，无法批量处理
  • 量价信号较弱：形态分析偏定性，缺乏操作流程那种量化触发机制
  • 无卖出扫描体系：止损止盈靠人工执行，无每日自动扫描
  • 无锁定期机制：缺乏3个月锁定等强制持有纪律
  • 无趋势保护线：卖出信号融入三维止损，但不如分档趋势保护精细
  • 无决策日志：结论不可追溯，难以复盘纠错
  • 无市场状态判定：不自动根据大盘环境调整操作偏向
  • 主观性强：不同人用同一框架可能得出不同结论

10互补使用建议

两种策略并非互斥，而是互补关系

经过全维度对比，两种策略各自的优势区域几乎不重叠，形成天然的互补结构：

推荐组合使用方式

环节用操作流程用分析框架

一句话总结

操作流程是"运营体系"，分析框架是"分析工具"。前者解决"如何在5000只股票里系统化地发现、跟踪、买卖"，后者解决"如何把一只股票彻底分析清楚并做出有纪律的决策"。实战中，用操作流程建立核心估值合格池并执行每日扫描，当某只股票触发买入候选时，再用分析框架做一次五步深度研判和风险矩阵梳理，最后用操作流程的量价信号和仓位规则执行交易、用卖出扫描体系管理持仓——这是两种策略融合使用的最佳路径。
