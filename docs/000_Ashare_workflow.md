# A股选股-估值-量价操作流程

## 0. 任务路由（agent 入口）

收到"按照本流程执行/分析…"类指令时，先在此表定位模块，再按对应章节的执行规则与脚本操作。**术语消歧**："值得关注的公司"是阶段一的 `worth_attention`（公司质量结论）；"今日值得关注的股票/走势"指每日扫描的**买入候选**（§8）。含"今日/走势/盘面/买点"的指令默认路由到 §8；"发起当日的每日扫描"类整体指令路由到 §9.1 组合流程（先持仓、后池）。

| 指令类型（示例） | 模块 | 脚本（确定性部分） | 主要输入 → 输出 | 大模型职责 |
| --- | --- | --- | --- | --- |
| 发起当日的每日扫描（持仓健康度 + 池内关注对象，支持盘中发起） | §9.1 组合流程 | `scan_holdings_sell_signals.py` → `screen_daily_volume_price_signals.py` | 持仓 + 估值池 → §9.2 组合摘要回复 + `000_daily_scan_log.md` 置顶条目 | 健康度红黄绿分组、关注对象筛选与关注原因、口径标注与收盘定型 |
| 分析今日关注股票的走势 / 找今日买入候选 | §8-§11 | `screen_daily_volume_price_signals.py --as-of` | `a_share_core_valuation_pool.csv` → `daily_buy_candidates.csv` + 扫描日志条目 | §10 人工核对当日公告/催化，定操作偏向 |
| 这只候选能不能下单 / 买入前决策 | §10 买入前闸门 | 无（模型+人工逐项填卡） | buy_candidate → `pretrade_decisions.csv` 一行 | 11 项闸门逐项核对，任一缺失不得建仓 |

| 对 worth_attention 公司做 L1-L5 分层 | §5.7-§5.8 | 无（模型判断） | triage → `a_share_watchlist_quality_tiers.csv` | 逐家分层与理由 |
| 估值排雷 / 更新核心估值合格池 | §6 | `build_a_share_core_valuation_pool.py` | L1-L4 + 估值表 → `a_share_core_valuation_pool.csv`（分层×估值准入矩阵 §6.2.1） | 分策略（A-G）估值档位判定 |
| 财报披露后的滚动更新 | §7 | `build_report_update_queue.py` | triage/tiers/pool → 更新队列 | 触发复核的质量/估值判断 |
| 单家公司值不值得关注 | §5.4 + 防过度纳入清单 + 边缘判例集 | 无 | → triage 行 + 决策日志 | 独立判断（轻量初筛） |
| 历史回放验证 | §12 | `backtest_signal_replay.py` | 点位时点池 → replay CSV | 解释成败、给调整建议 |
| 这只股票现在能不能买（池外单票快速研判） | 单票 express：§5.4 → §5.7 → §6 → §8 | 无（量价按 §8 规则单票核对） | 单票 → 各段产物 + 决策日志 | 全链路分段判定，任一段否决即止 |
| 指令命中重蹈覆辙红线（无信号左侧买入 / 摊平加仓 / 下调、取消割肉价 / 拆除防护层、整体转左侧） | 个人体系 §17 + 本文件 §15.1 | 无（模型对照红线清单逐项核对） | 用户指令 → 劝阻答复 + 替代路径；坚持执行则 §10.4 off_system 前置记录 + `discipline_intervention` 日志 | 红线识别、引用病历纠正劝阻、如实留痕 |

**单票按需全链路研判（express）**：对任意指定股票（含池外），按 §5.4 三类判定 → 若 `worth_attention` 按 §5.7 分层 → 若 L1-L4 按 §6 估值定档（入池按 §6.2.1 分层×估值准入矩阵） → 若入池再按 §8 检查当日量价信号，任一段否决即停止并给出结论。express 使用与批量流程完全相同的标准，不得降低门槛，也不得跳过 §10 人工核对与 §11 风险过滤；各段结论照常写入对应产物文件与决策日志，使一次性分析可追溯、可复用。该路径用于"今天想知道某只股票值不值得买"的快速决策场景。

所有结论按 §2.1 写入决策日志；执行分工与禁改标准见 §15。

## 1. 目标

本流程用于固定“全市场质量筛选 -> 核心公司估值排雷 -> 每日右侧量价触发 -> 买入候选输出”的可重复执行体系。

核心原则：

1. 业务质量决定是否值得长期跟踪。
2. 估值分析用于排除过度透支的股票，而不是只寻找显著低估股票。
3. 量价信号只在高质量、估值合格的股票池内触发。
4. 每日输出的是买入候选，不是自动买入指令。
5. 每一次业务质量、估值、量价、买入候选、移出或降级结论，必须同步写入 `data/processed/a_share_workflow_decision_log.csv`，保留简要信息来源、结论、结论时间、输入文件、输出文件、数据口径和判断理由。

## 3. 核心概念

**值得关注公司**：
通过业务质量筛选后，值得持续跟踪的上市公司。

**临界待定公司**：
两种情形之一：缺少可靠资料、暂时无法明确其实际水平且无明显负面（含次新股披露不足）；或可判断但当前缺乏持久优势，但所处行业并非结构性绝望，未来可能因新产品、客户验证、技术路线、行业格局、资产重组或经营改善而重新获得复核资格。临界待定公司不进入每日量价扫描。

**垃圾公司**：
永久业务排除公司。只给**坐实的**造假/治理灾难或结构性绝望行业；公司层面“当前无护城河但行业并非绝望”属于临界待定，不判垃圾公司。垃圾判定可在任何轮次由坐实证据产生（不限于第一轮），但绝不由质量分层的 L5 表达。进入该类后，后续季度不再分析其业务是否改善，不再进入质量分层、估值排雷和每日量价扫描，只维护证券基础信息；唯一例外是 §5.4.7 的窄纠错路径（原判证据被推翻，或规则 7 的控制权变更且责任人出清）。

**质量分层**：
在第一轮三类初筛之后，只对 `worth_attention`（值得关注）公司执行。只根据护城河、业务类型、竞争优势可持续性、技术壁垒、资本复制难度、市场地位、现金流质量和不可替代性分为L1-L5，不考虑当前股价。L1-L4 是保留在值得关注内的质量档；L5 不是保留档，而是“不再值得长期关注”的判定，将该公司由 `worth_attention` 降级为 `boundary_pending`，等待未来硬触发再复核。`garbage` 是三类初筛层面的判定（可在任何轮次因坐实证据产生，见 §5.4.7），不进入质量分层，也不用L5表达。

**核心质量公司**：
质量分层为L1或L2的公司。

**核心估值合格池**：
L1-L4 值得关注公司中，按分层×估值准入矩阵（§6.2.1）估值合格的股票，进入每日量价扫描。分两层：核心层（L1/L2，承载核心/准核心仓）与战术层（L3/L4，只允许战术仓）。层级越低准入越严——L1 合理估值（中性，甚至可接受较高估）即可入池，L4 仅低估且赔率达标才入池。一律排除高估、无法估值、极端bull case已被定价的股票。

**买入候选**：
核心估值合格池中出现有效右侧量价信号的股票。

## 4. 总流程

```text
交易所最新A股证券名单
  -> 更新 data/raw/a_share_securities.csv
  -> 第一轮三类初筛
  -> 剔除垃圾公司
  -> 值得关注公司和触发复核的临界待定公司
  -> 季度质量审查
  -> A股值得关注公司列表
  -> L1-L5质量分层
  -> L1-L4全量估值排雷（分层×估值准入矩阵）
  -> 核心估值合格池（核心层L1/L2 + 战术层L3/L4）
  -> 每日量价扫描
  -> 买入候选输出
  -> 人工核对公告、研报和当日催化
  -> 决定是否建仓、等待回踩或放弃
  -> 建仓后写入持仓清单
  -> 每日持仓监控与卖出扫描
  -> 持仓动作：持有/加仓资格/减仓/止盈/清仓/风险预警
```

## 5. 阶段一：季度全市场质量审查

### 5.1 执行频率

每个季度报告披露周期后执行一次完整更新；在财报季内，按单家公司报告披露进度滚动更新。

### 5.2 输入

1. `data/raw/a_share_securities.csv`
2. 上一版 `data/processed/a_share_attention_triage.csv`
3. 上一版 `data/processed/a_share_watchlist_quality_tiers.csv`
4. 公司最新年报、半年报、季报、业绩预告、业绩快报
5. 公司公告、交易所问询函、投资者关系材料
6. 权威研报和专业机构研究材料
7. 同行业历史校准文件和既有L1/L2/L3参考公司

### 5.3 季度开始先更新A股证券名单

每个季度开始分析前，先基于交易所公开名单刷新A股证券基础名单，确保新上市公司进入第一轮初筛。

```bash
python3 scripts/fetch_a_share_universe.py \
  --output data/raw/a_share_securities.csv
```

执行要求：

1. 保留 `source_provider`、`source_dataset`、`source_retrieved_at_utc`、`source_url` 等来源字段。
2. 脚本同时把本次名单另存为 `data/raw/snapshots/a_share_securities_YYYYMMDD.csv`（按检索日期命名，不可变，ADR-0001）；主路径覆盖为最新版。对比上一份快照识别本季度新增上市公司。
3. 新上市公司进入 `attention_triage`，但上市时间过短、披露材料不足时，不直接归为垃圾公司，应先归为 `boundary_pending`。
4. 更新证券名单这一动作不构成投资结论，但后续由此产生的新上市公司初筛结论必须写入 `data/processed/a_share_workflow_decision_log.csv`。

### 5.4 第一轮三类初筛

第一轮初筛只回答一个问题：这家公司以后是否值得继续消耗研究精力。它是轻量、全覆盖的第一层，对每一家A股给出 `worth_attention`、`boundary_pending` 或 `garbage` 之一。

#### 5.4.1 总体原则

1. **逐家独立判断**：每家公司单独分析，禁止用单一指标对多家公司批量归类。同业对比可作为判断的输入，但不能取代逐家判断。
2. **以竞争力为准**：基于公司实际竞争力判断，不以盈利水平或市值高低为主要依据。股价上涨、题材热度本身既不是入选证据，也不是排除证据。
3. **轻量初筛**：第一轮只判断业务模式、是否具备明显且可持久的竞争优势、是否存在明显负面；不要求逐家通读年报，明显垃圾可快速判定。通读年报、研报等深度分析留到对 `worth_attention` 做 L1-L5 分层和权威深度复核。
4. **入选从严**：`worth_attention` 的优势必须能通过资本复制测试——资金充足的新进入者难以快速复制并侵蚀其回报；“一定程度的护城河”须收紧为“持久、难以复制”。不设数量目标，数量由严格标准自然决定；存疑则不放入 `worth_attention`。
5. **排除分两种**：`garbage` 是永久排除，只给造假/治理灾难与结构性绝望行业；公司层面“当前无护城河但行业并非绝望”应放 `boundary_pending`（可逆），不要因一时平庸就永久排除。
6. **判不了就 boundary**：缺少可靠资料、无法判断实际水平、且无明显负面 → `boundary_pending`（含次新股披露不足）。

#### 5.4.2 三类定义

| 分类 | 定义 | 后续处理 |
| --- | --- | --- |
| `worth_attention` / 值得关注公司 | 具备能通过资本复制测试的持久优势：护城河、强市场地位、定价权与话语权；或竞争优势、政策优势、产业地位；或相对竞争对手有明确的差异化优势或整体优势。以竞争力为准，不看盈利/市值，从严。 | 进入质量分层（L1-L5）和后续估值排雷 |
| `boundary_pending` / 临界待定公司 | 两种情形之一：(a) 缺少可靠资料、暂时无法明确其实际水平、且无明显负面（含次新股披露不足）；(b) 可判断但当前缺乏持久优势，但所处行业并非结构性绝望，未来有发展或重获优势的可能。 | 不进入每日扫描；在出现可靠资料，或财报、公告、订单、客户验证、产品突破、重大转型、行业结构变化等硬触发时复核 |
| `garbage` / 垃圾公司 | 永久业务排除。只给：① 造假/治理灾难；② 结构性绝望行业。 | 永久排除，不再进入季度质量审查、质量分层、估值排雷和每日量价扫描，只维护证券基础信息 |

#### 5.4.3 进入 garbage 的条件

只在以下情形判为 `garbage`（永久排除）：

1. 造假/治理灾难（须坐实）：财务欺诈、重大虚假陈述、严重监管处罚、控股股东资金占用、严重审计异常，或管理层造假/严重不诚信——以行政处罚决定、刑事立案/判决、审计否定或无法表示意见、交易所纪律处分等权威文书为准。
2. 治理/财务长期不透明：现金流、应收、存货、负债、治理结构长期无法解释。
3. 结构性绝望行业：所处行业或细分本身低壁垒、不可能形成持久护城河——一旦证明能赚钱，资金充足的新竞争者可以轻易进入并迅速替代，行业内没有任何公司能建立持久优势。

**证据等级（governance_fraud 必填）**：`garbage(governance_fraud)` 行必须在 `evidence_basis` 标注证据等级——`坐实`（有权威文书）或 `需核验`（依据公开报道/历史记忆，待对照文书）。`需核验` 的垃圾判定必须在下一个季度审查周期内对照公告/处罚文书确认；确认不了坐实证据的，降回 `boundary_pending` 并记 `garbage_review` 日志。**仅有造假嫌疑（长期蹭热点、公告与财报明显不匹配、夸大宣传）而无坐实文书的，判 `boundary_pending` 并写明核验触发条件，不判 `garbage`**——嫌疑类公司在嫌疑坐实前不进入每日扫描，实际风险由 §11 风险过滤器第 7 条兜底。

注意：公司层面“当前同质化、暂无护城河”，若行业并非结构性绝望，应放 `boundary_pending`，不判 `garbage`；“被更强同行全面覆盖”属于后续分层与同业校准的考量，不在第一轮判 `garbage`。

`garbage` 一旦坐实即永久排除，不因股价上涨、短期利润改善、媒体热度、主题炒作或低PE重新进入研究队列。只维护证券基础信息。唯一纠错路径见 §5.4.7：原判证据被权威信息推翻，或符合规则 7（治理灾难发生在前控股股东、控制权已变更且责任人出清）。

#### 5.4.5 行业校准锚

第一轮标准通过逐行业判例校准；每个行业的校准结论以可追溯方式记录在 `docs/peer-group-calibration/` 下对应文件，并在此沉淀可推广规则。

白酒行业校准（见 `docs/peer-group-calibration/a-share-baijiu.md` §1.10）得到的可推广规则：

1. **护城河需穿越周期验证**：`worth_attention` 要求护城河持久且已在完整下行周期中验证；护城河真实但穿越下行周期的能力尚未验证 → `boundary_pending`（可逆，验证后可升）。
2. **行业增长前景不是第一轮排除项**：行业长期增长走弱不在第一轮排除公司，只在 L1-L5 分层与估值中折价；第一轮以竞争力为准。
3. **传承品牌即资本难复制的护城河**：品牌 + 产区/原产地 + 老窖/不可替代实物资产 + 基酒/陈年库存 + 深耕渠道是典型持久护城河，此类行业并非结构性绝望。
4. **ST/困境的处理**：仅因连年亏损被 ST/困境、未坐实造假 → 不判 garbage，按竞争力判（通常 boundary）；`garbage(governance_fraud)` 需有坐实的造假/治理灾难，并核验公告。
5. **多元化拖累**：核心业务护城河真实、但被亏损非核心业务拖累、可比性差 → `boundary_pending`（剥离/重组后可逆），不直接 worth_attention 或 garbage。

面板行业校准（见 `docs/peer-group-calibration/a-share-display-devices.md` §1.5）补充：

6. **资本周期型制造业的护城河**（面板、半导体代工、光伏、LED、锂电等强周期、资本密集行业）：若龙头优势建立在**不可逆的结构性整合 + 客观巨大的资本/规模/技术壁垒**上（替代在客观上已很难），可判 `worth_attention`，即便穿越周期盈利尚未充分验证；周期性在 L1-L5 分层与估值折价。与规则 1 的区分：护城河**持久性本身存疑**（可能在需求下行中被侵蚀）时仍需穿越周期验证 → boundary；壁垒是客观结构性的则不需此前置证明。
7. **治理灾难的可逆性**：治理灾难发生在**前控股股东**、现已更换控制权并重整 → `boundary_pending`（控制权变更=复核触发），不判永久 garbage；`garbage(governance_fraud)` 留给未解决/仍在持续、或同一责任人仍在位的造假与治理灾难。

建筑装饰行业校准（见 `docs/peer-group-calibration/a-share-decoration-construction.md` §1.5）补充：

8. **结构性绝望行业（structural_industry garbage 的适用）**：当行业商业模式结构性破损——低进入门槛、垫资/应收/强地产或单一客户依赖、优势 execution-heavy 而非持久护城河、历史规模或当前利润不代表持久回报——判 `garbage(structural_industry)`，**且适用于相对龙头**（execution、品牌、历史规模在破损模式里不构成持久护城河）。仅当存在**结构确实不同的更高壁垒子赛道**（如高端幕墙系统 vs 普通室内装修）才碗出到 `boundary`。结构性判决在子行业/子赛道粒度上做。

第一轮全量执行中校准补充：

9. **基础设施垄断**（机场、港口、高速、水务、燃气等）：有不可复制的区域垄断地位，但回报受管制、无定价权、特许期有限，护城河不转化为超额回报 → `boundary_pending`（可逆）；出现非航/免税/提价/资产注入等明确改善再升 `worth_attention`。
10. **极端牌照但回报受管制（核电等）**：即使牌照壁垒极强（如核电全国仅 2-3 家运营商），只要回报受管制、无定价权、护城河不转化为超额回报 → `boundary_pending`（规则 9 的延伸）；电价市场化、资产注入或非管制收益占比提升再升 `worth_attention`。
11. **银行**：只有穿越周期的优质 franchise（持续低成本存款、风控、显著高于同业的 ROE，如宁波银行）才判 `worth_attention`；普通、同质化、强周期或主要靠规模的银行 → `boundary_pending`。
12. **品类垄断与 capped-capacity 商品**：防过度纳入清单不能误伤两类真实护城河：
    - 细分品类龙头若已证明**品类垄断/全球领先 + 高毛利/高ROE/定价权 + 穿越周期超额回报**，不按“普通细分件”降级，仍可判 `worth_attention`（如美亚光电色选机）。
    - 商品行业若存在**政策性产能红线/指标稀缺 + 低成本曲线 + 一体化规模优势**，产能指标可视为类资源牌照，低成本一体化龙头可判 `worth_attention`；仅有产能、仅有成本优势或无定价权的普通周期公司仍判 `boundary_pending`。

#### 防过度纳入清单（worth_attention 的否决项）

worth_attention 从严。出现以下情形，默认 `boundary_pending`，不纳入 worth_attention：

1. **“研究价值/值得理解”不是理由**：worth_attention 要求持久、难复制、能转化为超额回报的护城河；“值得理解 ≠ 值得观察”。
2. **护城河理由自带对冲**：理由里出现“规模弱/偏小/需跟踪/研究价值/受益于景气/有望”等对冲措辞 → 默认 boundary。
3. **强位置 ≠ 超额回报护城河**：仅“细分/niche 能力强”，或强市场地位但属中游/低毛利/组装代工/客户高度集中 → 默认 boundary。
4. **资源类三要素**：需同时具备稀缺 + 低成本曲线 + 规模/储量；仅“有矿/有资源”不够 → 默认 boundary。
5. **景气受益不是护城河**：AI 资本开支、商品涨价、补贴等景气本身不构成护城河；要问景气退潮后是否仍有难复制壁垒。
6. **规制回报/无定价权**：含极端牌照（核电、规制公用）与普通银行业务 → boundary（规则 9/10/11）。
7. **与已判同行一致**：同行业/同类型中，更大或更强的同行若已判 boundary，更小的不应判 worth_attention。

豁免：若公司满足规则 12 的品类垄断或 capped-capacity 低成本龙头条件，不能仅因“细分”“周期”“成长空间有限”降为 `boundary_pending`；成长空间和估值吸引力留到 L1-L5 分层与估值阶段处理。

### 5.4.7 attention_class 状态迁移表

三类状态的全部合法迁移。任何迁移必须写决策日志，改判/纠错必须填 `supersedes_decision_id` 指向被替代结论。

| 迁移 | 触发条件 | 决策类型 |
| --- | --- | --- |
| `worth_attention` → `boundary_pending` | 质量分层判 L5（不再值得长期关注）；或复核发现持久优势证伪 | `quality_tier_review` / `attention_triage` |
| `worth_attention` → `garbage` | 坐实造假/治理灾难（任何轮次，见 §5.4.3 证据等级） | `attention_triage` 或复核 |
| `boundary_pending` → `worth_attention` | 硬触发（可靠资料、财报改善、订单、客户验证、产品突破、重组、行业结构变化）后复核通过 | `attention_triage` |
| `boundary_pending` → `garbage` | 坐实造假/治理灾难，或子行业被判结构性绝望 | `attention_triage` |
| `garbage` → `boundary_pending`（唯一纠错路径） | ① 原判所依据的证据被权威信息推翻（如处罚撤销、判决改判）；② 规则 7：治理灾难属前控股股东、控制权已变更且责任人出清；③ `需核验` 等级在核验期内确认不了坐实证据 | `garbage_review` |

补充约定：

1. `garbage` 不可直达 `worth_attention`；纠错只回 `boundary_pending`，再按硬触发正常复核。
2. `boundary_pending` 内区分两种子情形并写入 `boundary_recheck_trigger`：`insufficient_disclosure`（资料不足，含次新股）与可逆业务否决（可判断但当前缺乏持久优势）；两者复核触发不同（前者=资料可得，后者=业务硬触发）。
3. 结构性绝望行业（`structural_industry`）判决原则上不可逆；仅当公司实质转型脱离破损模式（对照汇绿生态/东方新能判例）或子赛道被证明结构不同，方可经 `garbage_review` 回 `boundary_pending`。
4. 质量分层（L1-L5）只在 `worth_attention` 内运作，永不产生 `garbage`。

### 5.6 进入季度质量复核队列的条件

满足任一条件即进入队列：

1. 新上市公司。
2. 最新报告期晚于上次质量审查日期。
3. 原L1/L2公司。
4. 原L3公司出现业务、订单、利润率、技术路线、行业景气明显改善。
5. `boundary_pending` 公司出现重大业务变化、重大订单、关键产品突破、资产重组、客户验证、行业结构变化或连续财报改善。
6. 毛利率、净利率、经营现金流、负债率、研发投入、收入增速、利润增速出现重大变化。
7. 发生重大诉讼、监管处罚、审计异常、控股股东风险、财务疑点。

以下公司不进入队列：

1. `attention_class = garbage` 的公司。
2. `boundary_pending` 但没有硬触发的公司。
3. 只有股价上涨、传闻、小作文或主题热度，但没有公告、财报、客户验证、订单、权威研报或行业数据支持的公司。

### 5.7.1 分层校准细则

（自原权威深度复核协议并入。）分层完成足够数量后整体应呈金字塔形：上层是稀缺关注名额，不是"公司还不错"的描述。

1. **L1 市值中性**：L1 留给能对抗现有最强锚点的核心候选；当前市值、指数权重、流动性不得作为 L1 的必要条件。
2. **L1 三种子类型**，避免混用理由：
   - 持久护城河/现金流锚：护城河极难复制、现金转化与利润质量卓越，即使行业增长放缓也可为 L1；行业增长弱只记入估值与仓位备注，不单独降档。
   - 战略瓶颈/平台锚：掌握关键技术、系统平台、资源、标准、客户认证层或供应链瓶颈，且资金充足的进入者难以替代。
   - 小市值例外：权威证据显示真正不可替代的利基、全球/全国领先地位与高切换成本时，小市值不封顶质量档位，只提高证据要求、并可能降低最终仓位。
3. **L2 不因估值降档**：L2 用于优势明确但存在周期、空间、瓶颈验证、治理或对比 L1 锚点位置等实质性折扣的公司；当前估值敏感性属于估值阶段，不得单独用于降质量档。
4. **L3 不是中间桶**：必须至少有一个明确的战术关注理由（行业催化、右侧触发候选、周期拐点、局部瓶颈、差异化技术、股东回报催化或有据可查的困境反转路径）；理由只是"公司不错/可能增长/可能反弹"则降 L4。
5. **L4 是保守默认档**：可理解、可偶尔复核，但护城河强度、催化清晰度、同业差异或证据质量不足以进前三层。
6. **锚点双重对照**：同业对照不仅比弱者，必须与已定 L1/L2/L3 锚点比较；不如现有 L3 锚点的不进 L3。
7. **医疗器械**：平台型 L1 需产品广度被共享医院渠道、装机保有量、服务网络、质量体系、全球注册能力、研发复用与现金转化共同强化，仅产品线多不够；电生理、血管介入、高端影像、内窥镜、植入物、耗材、生物材料等专科高壁垒公司默认从 L2 起，除非能独立证明品类定义级地位；被 L1 平台收购是技术验证但不自动升 L1，协同效应记为升档触发而非既成事实；产品广度稀释 ROIC、拖累现金转化或关键产品线丢份额时平台降档。
8. **AI/半导体链**：不得把短期利润爆发等同于更强护城河。光模块、PCB、封装、半导体设备、EDA、接口芯片等必须同时评估「当前瓶颈强度」与「穿越周期的技术/生态护城河」：瓶颈强但持久性弱者可作战略瓶颈锚 L1，但必须显式标注产品周期、客户集中与定价风险；护城河更深者不得仅因近期业绩弹性低而排在当前周期赢家之后。
9. **禁止捷径**：不得仅凭行业关键词、同业标签或历史分数派档；聚合站/F10 简介只作线索不作证据；不得仅凭当前盈利推断护城河；不得跳过资本复制测试；不得让当前估值决定档位。

### 5.8 质量审查执行规则

质量审查只判断业务质量，不考虑当前股价和估值。

输入材料：

1. 公司名称和股票代码。
2. 所属行业、同行组和主要竞争对手。
3. 最新年报、半年报、季报、业绩预告和业绩快报摘要。
4. 公司公告、交易所问询函、监管处罚、投资者关系材料。
5. 权威研报和专业机构研究材料。
6. 当前已有 `attention_class`、质量分层和策略标签。
7. 同行业L1/L2/L3参考公司。

执行步骤：

1. 说明公司主要业务、利润来源和业务结构。
2. 识别护城河来源：品牌、资源、技术、客户认证、成本、渠道、网络效应、监管牌照等。
3. 执行资本复制测试：判断资金充足的新玩家能否快速复制并超越。
4. 与同行比较，判断是否被更强公司全面覆盖，或是否具备差异化不可替代位置。
5. 判断当前业务优势是否可持续，并做前瞻侵蚀检验：列出未来 2-3 年最可能侵蚀护城河的因素（技术替代、政策/集采、携资进入、大客户或核心团队流失），评估概率与影响，写入 `moat_summary` 或 `tier_reason`。
6. 检查一票否决事项：财务造假、治理风险、现金流恶化、低壁垒高竞争、媒体炒作和造假嫌疑。
7. 先确认 `attention_class`：默认维持现状；仅当出现坐实负面（→`garbage`）或持久优势证伪（→`boundary_pending`）时按 §5.4.7 迁移表改判，并填 `supersedes_decision_id`。
8. 如果改判 `garbage`，必须写明垃圾分类原因与证据等级（§5.4.3），并记录后续永久排除。
9. 如果不是 `garbage`，再根据业务质量分为L1/L2/L3/L4；若业务质量只够L5（低壁垒、被更强同行全面覆盖等），不保留在值得关注内，降级为 `boundary_pending`。
10. 给出观察池动作：新增、维持、降级或移除。
11. 将来源、结论、时间、输入文件、输出文件和简要判断理由写入结论日志。

输出字段：

```text
security_code
security_name
attention_class
quality_tier
primary_strategy_tag
moat_summary
capital_replicability
peer_comparison
garbage_reason
boundary_recheck_trigger
tier_reason
watch_action
evidence_sources
decision_log_summary
```

## 6. 阶段二：值得关注公司估值排雷（L1-L4）

### 6.1 执行范围

对全部 L1-L4 值得关注公司执行估值分析（worth_attention 经从严初筛后总数少于300家，全量估值成本可控且避免 L3/L4 中的深度低估机会被系统性忽略）。分层只决定**买入准入门槛与仓位上限**（§6.2.1），不改变估值方法本身。

### 6.2 输出分类

| 估值档位 | 含义 | 是否进入核心估值合格池 |
| --- | --- | --- |
| 低估 | 当前市值明显低于保守或正常估值，赔率充分 | 按 §6.2.1 矩阵（L1-L4 均可） |
| 较低估 | 当前价格有明显吸引力，但存在周期、政策、业绩验证或行业折扣 | 按 §6.2.1 矩阵（L1-L3） |
| 中性 | 估值大致合理，未明显透支未来 | 按 §6.2.1 矩阵（L1/L2） |
| 可接受较高估 | 略贵，但bull case尚未完全定价，且后续仍可能上修 | 按 §6.2.1 矩阵（L1常备；L2仅事件通道） |
| 高估 | 已计入极端bull case，或需要过度乐观假设才能成立 | 否（任何层级） |
| 无法估值 | 缺少足够数据或关键假设不可验证 | 否（任何层级） |

### 6.2.1 分层×估值准入矩阵（买入门槛随质量层级递严）

质量越高，买入估值门槛越宽；质量越低，要求的安全边际越深。矩阵同时决定入池资格与该股票的仓位身份：

| 层级 | 可入池的估值档位 | 池内层 | 仓位身份与上限 | 附加买入门槛（§10 闸门） |
| --- | --- | --- | --- | --- |
| L1 | 低估 / 较低估 / 中性 / 可接受较高估（需盈利预测持续上修或明确催化，§6.6） | 核心层 | 低估/较低估≤10%总资产；中性/可接受较高估≤5%（§13 v20） | 赔率按个人体系默认（>2；稳定现金流 1.5-2） |
| L2 | 低估 / 较低估 / 中性 / 可接受较高估（仅事件通道*） | 核心层 | 低估/较低估≤10%总资产；中性≤5%；可接受较高估仅经事件通道，按突破/事件仓单票≤5%（计入战术合计） | 赔率按个人体系默认（>2；高波动 >3） |
| L3 | 低估 / 较低估 / 中性（仅事件通道*） | 战术层 | 战术仓单票≤5%总资产（战术持仓合计 ≤15%） | 赔率 ≥2.5，且必须有明确战术理由兑现路径 |
| L4 | 仅低估 | 战术层 | 极小战术仓：单票 ≤2.5%总资产，不加仓，计入战术仓合计 | 赔率 ≥3，且低估结论须经归一化口径复核（防一次性利润粉饰） |

执行说明：

1. 矩阵是**买入准入**，不是卖出指令：已持仓股票估值档位滑出准入范围时按 §14 风险预警 5 复核，不自动清仓。
2. L3/L4 的战术层股票出现买入信号时一律先经 §10 人工闸门（§8.10 优先级 C/D），不允许仅凭脚本信号建仓。
3. 分层或估值档位任何一侧变化后，重跑池物化脚本即自动按矩阵重算入池资格。
4. **事件通道（v24修订，watch_only 信号通道：L2×可接受较高估、L3×中性）**：两组合不具备常备买入资格，在池文件中为 `watch_only` 仅观察层。通道激活条件（当日同时满足）：①§8.7信号分级=强，且含突破确认项（放量突破前高/关键均线/平台二次确认之一）；②当日完成单票express估值复核，确认档位未滑入高估、且未触发§8.8过度延伸过滤（或符合其例外条款）——L3×中性按D/爆发前瞻口径仍≤bull case合理市值100%；L2×可接受较高估确认该档位定义仍成立（§6.6：盈利预测持续上修或明确催化，如D口径≤bull case合理市值120%且仍在上修、GARP归一化PEG≤2.0且持续上修）；③通过§10闸门，赔率按层级门槛（L2按个人体系默认>2、高波动>3；L3≥2.5）。满足后允许按§13战术上限（单票≤5%总资产）一次建仓，割肉价按daily档启动结构口径设定。**催化剂不作为买入前置**（公告级信源常滞后于股价，等公告再买即追高——2026-07浪潮预告后+31%追入 vs 紫光7/8破前高先行的对照），改为**24小时验证义务**：建仓后24小时内完成个人体系§9.4异动响应，核实真实驱动（公告、订单、行业数据、板块级联动或权威研报之一，注明信源等级）；核实不到→禁止加仓、割肉价收紧至启动日低点/启动平台口径、决策日志记`catalyst_missing`；驱动被证伪或确认仅题材/小作文→按个人体系§9.5立即退出，且不得转为长期仓（个人体系§17.3 R6）。L1全部非高估档位本就常备可买（矩阵首行），无需通道；L4不适用事件通道；高估/无法估值任何层级一律不适用（估值排雷底线）。
5. **watch_only 仅观察层（v20）**：L1-L4中估值为低估/较低估/中性/可接受较高估但未过本矩阵的组合（当前主要为L2×可接受较高估、L3×中性、L4×非低估），物化进池文件供每日扫描可见，不具备常备买入资格（L2×可接受较高估、L3×中性经执行说明4通道激活除外）；高估/无法估值不物化。

### 6.3 估值原则

1. 估值只用于排除过度高估的核心质量公司。
2. 不因单纯PE高而排除处于产业爆发、研发投入期、盈利爬坡期的公司。
3. 不因单纯PE低而认定低估。
4. 对一次性亏损、一次性收益、研发投入、股权投资收益、公允价值变动、资产减值、补贴、会计处理变化进行归一化处理。
5. 对周期股使用中枢利润、资源NAV、商品价格中枢，不使用周期顶部单年利润。
6. 对产业链爆发股使用2-3年正常化利润和需求、瓶颈、利润三张票，不机械使用TTM PE。
7. 对早期高研发公司使用收入质量、毛利率成熟路径、研发效率、客户验证、产品壁垒和未来正常化利润，不机械使用当前亏损。

### 6.4 特殊估值失真处理

出现以下情形时，必须使用归一化估值：

1. 单季度或单年度存在一次性亏损。
2. 单季度或单年度存在一次性投资收益、公允价值收益或资产处置收益。
3. 公司处于研发投入期，当前利润被研发费用、产能爬坡、销售投入压低。
4. 公司处于行业周期底部，当前利润低于中枢利润。
5. 公司处于产业爆发前夜，当前利润尚未体现订单、产能、技术路线或客户验证。
6. 公司处于商业模式转换期，旧业务拖累利润，新业务尚未充分释放。

归一化估值字段：

```text
normalized_profit = 扣非净利润 - 一次性收益 + 一次性亏损调整 + 合理研发/产能爬坡归一化调整
normalized_margin = 成熟业务合理净利率
normalized_revenue = 可验证订单、产能、行业需求支持下的未来2-3年收入
normalized_market_cap = normalized_profit * 合理估值倍数
```

收入质量三分类（归一化的第一步）：

| 收入类型 | 示例 | 处理 |
| --- | --- | --- |
| 经常性收入 | 产品销售、稳定服务费、可持续订单 | 计入经常性利润，可外推，适用正常估值倍数 |
| 一次性收入 | BD授权首付款、资产处置、大额补贴、公允价值变动 | 从经常性利润中剔除 |
| 里程碑/或有收入 | BD后续里程碑款、对赌兑现 | 不并入经常性利润，按兑现概率作为期权价值单独估算 |

```text
经常性PE = 当前市值 / 经常性利润
```

一次性收入占当期利润比例高的公司，必须用经常性PE替代表观PE参与档位判定，防止被一次性利润粉饰的低PE误导。

### 6.5 分策略估值方法

#### A：DCF / 现金流复利型

指标：

1. PE历史分位。
2. 自由现金流收益率。
3. 股息率和回购注销率。
4. ROE/ROIC稳定性。
5. 未来3年收入和利润增速。

预期年化收益：

```text
预期年化收益 = FCF收益率 + 保守利润增长率 + 股息/回购收益率 + 估值修复年化贡献
```

档位标准见 §6.6 总表。

#### B：烟蒂深度低估型

保守清算价值：

```text
保守清算价值 = 现金 + 0.8 * 应收账款 + 0.5 * 存货 + 0.3 * 固定资产 - 总负债
```

档位标准见 §6.6 总表。

适用说明：估值排雷只对 L1/L2 执行，而典型烟蒂公司很少具备 L1/L2 级护城河，B 类在核心估值合格池内近似空集。烟蒂机会实际主要出现在 L3 战术观察（§8.10 优先级C 人工复核路径）；本模型为该路径与个别 L2 特例保留。

#### C：GARP成长型

指标：

1. 未来3年利润CAGR。
2. 归一化PEG。
3. 收入质量和市场空间。
4. 毛利率、净利率、经营现金流转化。
5. 增长是否可防守。

档位标准见 §6.6 总表。

#### D：产业链爆发 / 关键瓶颈型

必须同时评估：

1. 需求票：下游需求真实增长。
2. 瓶颈票：公司处于供给瓶颈或关键不可替代环节。
3. 利润票：需求和瓶颈能显著转化为上市公司收入和利润。

未来合理市值：

```text
未来合理市值 = 未来2-3年正常化净利润 * 合理PE
```

档位标准见 §6.6 总表。

#### E：落难白马型

指标：

1. 核心业务是否被结构性破坏。
2. 利润预期是否停止下修。
3. 现金流是否健康。
4. 管理层资本配置是否改善。
5. 估值是否低于正常化盈利水平。

档位标准见 §6.6 总表。

#### F：垄断资源型

指标：

1. 资源储量和NAV。
2. 成本曲线位置。
3. 商品价格中枢。
4. 穿越周期现金流。
5. 扩产能力和资本纪律。

档位标准见 §6.6 总表。

#### G：股东回报型低估

指标：

1. PB/ROE匹配度。
2. 分红率。
3. 股息率。
4. 回购注销率。
5. 资产质量。

档位标准见 §6.6 总表。

### 6.6 估值审查执行规则

估值审查只对L1/L2核心质量公司执行，目标是排除过度高估，不是只寻找显著低估股票。估值必须先确定策略标签，再使用对应模型。

输入材料：

1. 公司名称、股票代码、质量分层和策略标签。
2. 最新价格、市值、PE、PB、股息率、回购率和历史估值区间。
3. 最新财报核心指标：收入、扣非净利润、经营现金流、毛利率、净利率、ROE/ROIC、资产负债率、应收、存货、资本开支。
4. 未来3年一致预期和权威研报核心假设。
5. 行业周期位置、商品价格中枢、产业链供需状态或渗透率阶段。
6. 一次性收益、一次性亏损、研发投入、产能爬坡、周期底部、商业模式转换等估值失真因素。
7. 当前市值与bull case合理市值、正常化利润估值或资源NAV的差距。

执行步骤：

1. 确认适用估值方法：DCF/现金流复利、烟蒂深度低估、GARP、产业链爆发、落难白马、垄断资源、股东回报型低估。
2. 适用性前置检验：先确认策略标签仍然成立（增长被证伪的GARP、把一次性利润当成长的公司应重贴标签），再确认所选模型对该公司当前状态有效。典型失效场景：PE/PEG用于周期股、亏损公司或一次性收入占比高的公司；PB用于轻资产公司；PS用于利润率无改善路径的公司；DCF用于现金流为负或极不稳定的公司。方法失效时改用替代模型或归一化口径；无可用方法则判"无法估值"。
3. 判断当前PE/PB/PEG是否失真；如失真，使用归一化估值（含收入质量三分类与经常性PE）。
4. 写出估值模型关键输入，例如归一化利润、未来2-3年利润CAGR、合理PE、NAV、中枢FCF收益率、股息率或回购率。
5. 反向预期拆解：从当前市值反推市场隐含的增速、利润率或永续假设，与自己的关键输入对比，明确"市场已定价什么、预期差在哪"；隐含假设是否已达到bull case水平，是 `bull_case_priced_in` 的判定依据。
6. 给出bull case核心假设，并判断当前价格是否已经反映bull case。
7. 分为：低估、较低估、中性、可接受较高估、高估、无法估值。
8. 给出合理价区间 `fair_price_low`/`fair_price_high`：该股按对应策略模型处于"中性"档的价格带（如 D 档为 bull 合理市值×80%-100%、C 档为归一化 PEG 1.0-1.5 对应价格），换算依据写入 `fair_price_basis`；区间必须与所定档位自洽（低估的估值价应低于区间下沿，高估应高于区间上沿，中性应落在区间内），锚点数据不足时按档位标准带换算并在 basis 注明。
9. 判断是否进入核心估值合格池。
10. 给出等待右侧触发时需要观察的关键指标。
11. 将来源、结论、时间、输入文件、输出文件和简要判断理由写入结论日志。

估值模型和档位：

| 策略类型 | 模型 | 低估 | 较低估 | 中性 | 可接受较高估 | 高估 |
| --- | --- | --- | --- | --- | --- | --- |
| DCF/现金流复利型 | 预期年化收益 = FCF收益率 + 保守利润增长率 + 股息/回购收益率 + 估值修复年化贡献 | >=12% | 8%-12% | 6%-8% | 4%-6%，且L1或有明确催化 | <4%，或PE处历史90%分位以上且增长放缓 |
| 烟蒂深度低估型 | 保守清算价值 = 现金 + 0.8*应收账款 + 0.5*存货 + 0.3*固定资产 - 总负债 | 市值 <= 清算价值*0.7且有催化 | <=0.9且资产真实 | 接近清算价值但催化不足 | 不适用 | 资产真实性不足、负债压力大或无兑现路径 |
| GARP成长型 | 归一化PEG | PEG <=0.7且增长可防守 | 0.7-1.0 | 1.0-1.5 | 1.5-2.0且盈利预测持续上修 | >2.0且无持续上修证据 |
| 产业链爆发/关键瓶颈型 | 未来合理市值 = 未来2-3年正常化净利润 * 合理PE | 当前市值 <= bull case合理市值*60% | 60%-80% | 80%-100% | 100%-120%且盈利预测仍在上修 | >120%或bull case依赖极端假设 |
| 落难白马型 | 正常化利润和修复后合理PE | 正常化PE显著低于历史中枢且困境边际改善 | 存在明显修复空间 | 公司质量仍在但触底不足 | 不适用 | 结构性恶化或修复已定价 |

| 股东回报型低估 | PB/ROE、股息率、回购注销率、资产质量 | 股息率+回购率显著高于无风险利率且PB未透支ROE | 股东回报清晰且估值低于资产质量 | 稳健但弹性有限 | 不适用 | 低增长但估值高于回报能力 |

输出字段：

时点字段口径：`valuation_reviewed_at` 为本次估值结论产生日（复核触发比较基准，区别于核心池物化日 `pool_as_of`）；`valuation_price_as_of` 为估值所用价格/市值的交易日；`evidence_available_at` 为所用最新财务资料的公开可得日（公告日，非报告期末）。估值结论没有固定有效期，失效由 §7 复核触发与 §7.5 买入冻结控制。

### 6.7 核心估值合格池物化脚本接口

估值审查输出的是估值结论，脚本只负责把已审查完成的估值表物化为每日扫描输入，不重新创造估值意见。

```bash
python3 scripts/build_a_share_core_valuation_pool.py \
  --valuation data/processed/a_share_focus_watchlist_l1_l2_valuation.csv \
  --tiers data/processed/a_share_watchlist_quality_tiers.csv \
  --output-csv data/processed/a_share_core_valuation_pool.csv \
  --output-md data/processed/000_a_share_core_valuation_pool.md \
  --log-file data/processed/a_share_workflow_decision_log.csv \
  --as-of YYYY-MM-DD
```

要求：

1. 纳入 L1-L4，按 §6.2.1 分层×估值准入矩阵判定入池资格，并输出 `pool_layer`（core=L1/L2，tactical=L3/L4）。
2. 估值档位为高估、无法估值的任何层级一律排除。
3. 每个纳入核心估值合格池的结论都要写入结论日志。
4. 估值表若含 `valuation_reviewed_at` 字段，物化时原样透传到池文件；`pool_as_of` 只表示物化日，不得当作估值复核日使用（§7.3 复核触发以 `valuation_reviewed_at` 为准，缺失时才回退 `pool_as_of`）。
5. 未过准入矩阵但估值为低估/较低估/中性/可接受较高估的 L1-L4 公司，输出 `pool_layer = watch_only` 仅观察层（v20）：进入每日扫描可见范围，不具备买入资格；watch_only 行日志 `decision_type = scan_watch_only`。
6. `fair_price_low`/`fair_price_high`/`fair_price_basis` 原样透传到池文件；阅读版 MD 对每只股票（含排除名单）展示合理价区间与空间（区间中值相对估值价的涨跌幅），不得只列当前股价。

### 7.4 事件复核触发

出现以下事件时立即进入事件复核队列：

1. 业绩预告或业绩快报大幅超预期或低于预期。
2. 重大订单、重大客户认证、重大产品发布。
3. 重大并购、资产出售、控制权变更。
4. 交易所问询、监管处罚、审计异常。
5. 行业政策、技术路线、商品价格或竞争格局发生重大变化。

### 7.5 复核期买入冻结（buy_blocked）

估值复核触发（§7.3）或事件复核触发（§7.4）后、复核完成前，该股票处于 `review_pending` 状态并冻结新增买入：

1. 队列脚本对 `valuation_review_needed` 的股票输出 `buy_blocked = review_pending`。
2. 每日量价扫描读取更新队列（`--review-queue`），对池内处于冻结状态的股票即使出现有效信号也只输出 `buy_blocked_review_pending`，不列买入候选（§8.9/§11）。
3. 冻结不影响已有持仓的卖出扫描与风险监控。
4. 解除条件：完成质量/估值复核并更新 `valuation_reviewed_at` 后，重新生成更新队列即自动解除；复核结论若为高估/无法估值或降档，则按正常规则移出核心池。
5. 预告季时效（v20）：L1-L4名单内公司披露业绩预告/快报当晚，须在下一交易日开盘前完成单票 express 估值增量更新（含策略标签复检，如GARP拐点确认改挂D口径），使买入冻结不横跨信号日；boundary_pending 公司命中复核触发的，当日写入复核队列。

MACD：

```text
fast = 12
slow = 26
signal = 9
```

成交量指标：

```text
vol_ma5 = 5日平均成交量
vol_ma20 = 20日平均成交量
vol_ma60 = 60日平均成交量
vol_baseline_60_21 = 第60日至第21日前的平均成交量
day_vol_ratio_20 = 当日成交量 / 前20日平均成交量
vol_3d_ratio_20 = 近3日成交量合计 / (3 * 前20日平均成交量)
vol_5d_ratio_20 = 近5日成交量合计 / (5 * 前20日平均成交量)
vol_5d_ratio_baseline = 近5日平均成交量 / vol_baseline_60_21
vol_percentile_120 = 当日成交量在近120日成交量中的分位
amount_ma20 = 20日平均成交额
```

### 8.5 有效放量定义

满足任一条件即可认定为有效放量：

1. 单日放量：
   ```text
   day_vol_ratio_20 >= 1.8
   ```

2. 连续放量：
   ```text
   vol_3d_ratio_20 >= 1.5
   或 vol_5d_ratio_20 >= 1.4
   ```

3. 启动前基准放量：
   ```text
   vol_5d_ratio_baseline >= 1.6
   且近5日至少3日成交量 > vol_baseline_60_21 * 1.2
   ```

4. 持续堆量：
   ```text
   vol_ma5 > vol_ma20 > vol_ma60
   且 vol_ma5 / vol_ma60 >= 1.5
   ```

5. 高分位放量：
   ```text
   vol_percentile_120 >= 80
   且近5日价格涨幅 > 3%
   ```

说明：连续多日放量会抬高20日均量，因此不能只用当日成交量/20日均量判断是否放量，必须同时使用启动前基准量、3日/5日合计量和成交量分位。


### 8.7 右侧信号

**核心原则**：只要出现**有效放量上涨**（满足 §8.5 有效放量且当日收涨），无论是否突破前高或关键均线，即列入 `buy_candidate`（仍受 §8.8 过度延伸过滤与 §11 风险过滤）。前高突破、均线突破、相对强度等不是入选的必要条件，而是**信号分级**的加分项。§8.7.1-§8.7.6 是**并列的独立触发形态**，各自满足即可触发 `buy_candidate`：其中 8.7.1/8.7.2/8.7.6 属于有效放量上涨的细分；8.7.3（缩量涨停）、8.7.4（连续上涨）、8.7.5（温和放量多头，量能可低于 §8.5 门槛）不要求满足 §8.5 有效放量，是显式例外。8.7.7（相对强度确认）不独立触发，只作为信号分级的突破确认项。

**信号分级**（按放量 × 短期多头 × 突破确认三层递进给推荐等级）：

| 信号分级 | 条件 | 默认操作偏向 |
| --- | --- | --- |
| 强 | 有效放量上涨 + 短期多头（§8.6 日线多头或准多头） + 突破确认（§8.7.1/8.7.2/8.7.6 之一或 §8.7.7 相对强度确认） | 可建目标仓位1/3 |
| 中 | 有效放量上涨 + 短期多头，暂无突破确认 | 可小仓试探或等确认 |
| 弱 | 有效放量上涨，短期趋势未转多 | 等确认 |

三档均保留 `buy_candidate` 状态；分级只决定推荐强度与仓位偏向，弱势市场（§8.12）下各档偏向再下调一档。

#### 8.7.1 放量突破前高

前高周期：

```text
60日、120日、250日、500日、750日
```

触发：

```text
close > 对应周期前高 * 1.005
且满足有效放量
且收盘价位于当日振幅上半区
```

收盘位置：

```text
close_location = (close - low) / (high - low)
要求 close_location >= 0.6
```

#### 8.7.2 放量突破关键均线 / 趋势启动

前高突破不是买入候选的唯一触发条件。高质量股票在主升早段可能先表现为放量突破关键均线、均线系统转多头和持续承接，然后才突破长期前高。

触发：

```text
close > MA20
且 close > MA60
且满足有效放量
且收盘价位于当日振幅上半区
且满足日线多头或准多头
```

准多头：

```text
close > MA5 > MA10 > MA20
且 close > MA60
且 MA20斜率走平或向上
```

强确认：

```text
日线多头
或 最近3-5个交易日内放量冲高回落后重新收强
或 个股相对行业指数和沪深300明显走强
```

收盘位置：

```text
close_location >= 0.6
```

适用说明：

1. 如果已突破前高，优先按`8.7.1 放量突破前高`处理。
2. 如果未突破前高，但放量突破MA20/MA60并形成准多头或多头排列，可进入`buy_candidate`或`wait_confirmation`。
3. 如果当日放量但收盘位置低于0.6，先进入`wait_confirmation`，等待后续1-5个交易日重新收强。
4. 如果价格仍低于MA120或MA250，优先级低于突破长期前高的信号，但不自动排除。

#### 8.7.3 缩量涨停

涨停阈值：

```text
主板普通股票：涨幅 >= 9.5%
创业板/科创板：涨幅 >= 19%
北交所：涨幅 >= 29%
ST股票：涨幅 >= 4.5%
```

缩量条件：

```text
当日成交量 <= vol_ma20 * 1.2
或 在前一日已经放量后，当日成交量 <= 前一日成交量 * 0.85
```

#### 8.7.4 连续上涨

触发：

```text
最近5日内至少4日上涨
且5日累计涨幅在8%-25%之间
且日线多头
且没有单日极端放量长上影
```

#### 8.7.5 温和放量多头

触发：

```text
日线多头
且20日涨幅在3%-20%之间
且vol_5d_ratio_20在1.1-1.8之间
且MACD在最近3个交易日内金叉
```

#### 8.7.6 平台突破后二次确认

触发：

```text
前期已放量突破
随后3-10个交易日横盘或缩量回踩
回踩不跌破突破平台2%以上
再次放量创新高
```

#### 8.7.7 相对强度确认

触发：

```text
个股近20日收益率 > 沪深300近20日收益率 + 8pct
且个股近20日收益率 > 所属行业指数近20日收益率 + 3pct
```

**脚本实现范围**：8.7.0 放量上涨、8.7.1、8.7.2、8.7.3、8.7.4、8.7.5 由 `screen_daily_volume_price_signals.py` 判定；8.7.6 平台二次确认与 8.7.7 相对强度确认需要突破历史状态与行业指数数据，暂由人工/大模型在 §10 核对时判定，脚本未实现。

### 8.8 过度延伸过滤

满足任一条件时，不直接列为买入候选，改为`wait_pullback`：

```text
close / MA20 - 1 > 25%
或 近5日涨幅 > 30%
或 近20日涨幅 > 60%
```

例外：

1. 同日或近期有明确财报、订单、客户认证、政策或产品催化。
2. 盈利预测出现重大上修。
3. 回踩平台不破后再次放量突破。

### 8.9 买入候选状态

| 状态 | 定义 |
| --- | --- |
| `buy_candidate` | 业务质量合格、估值未过度透支、出现有效放量上涨（无论是否破前高，含 §8.7 各右侧形态），并按 §8.7 信号分级标注 强/中/弱 |
| `wait_pullback` | 趋势强，但短期涨幅过大，等待回踩20日线或平台不破 |
| `wait_confirmation` | 单日异动明显，但收盘位置、持续承接或后续收强不足 |
| `wait_breakout` | 业务和估值合格，但尚未出现有效放量上涨 |
| `wait_repair` | 估值和业务合格，但跌破60日线或年线（MA250）未修复 |
| `liquidity_filtered` | 出现买入信号但 `amount_ma20` 低于流动性门槛（§11），不列为买入候选，仅提示 |
| `buy_blocked_review_pending` | 处于 §7.5 复核期买入冻结（财报/事件复核未完成），即使有有效信号也不列买入候选，仅提示 |
| `signal_watch_only` | watch_only 仅观察层出现有效量价触发（v20）：默认可见不可买，触发即走个人体系§9.4的24小时异动响应与§7.4事件复核。例外（v23/v24）：L2×可接受较高估、L3×中性触发**强信号**（含突破确认）时，当日可直接按事件通道（§6.2.1 执行说明4）核对——express估值复核+§10闸门通过即可建仓，催化剂验证义务后置24小时；其余组合仍须经复核入池后方可成为买入候选 |
| `inactive_watch` | 仍在核心估值合格池中，但无买入信号 |
| `remove_watch` | 业务thesis或估值基础被证伪 |

### 8.10 买入候选优先级

| 优先级 | 条件 |
| --- | --- |
| S | L1，低估/较低估/中性，放量突破120日以上前高，日线多头，长期趋势确认（§8.6），相对强度确认 |
| A | L1/L2，低估/较低估/中性，出现放量突破前高、放量突破关键均线、温和放量多头或平台后二次确认 |
| B | L1，可接受较高估，但盈利预测持续上修且出现强右侧信号 |
| C | L3（战术层，低估/较低估）出现强右侧信号：战术仓候选，必须先经 §10 闸门人工复核 |
| D | L4（战术层，仅低估）出现强右侧信号且赔率≥3：极小战术仓候选，必须先经 §10 闸门人工复核，默认操作偏向下调一档 |

watch_only 层不参与 S/A/B/C/D 定级与 §8.11 强势跟踪；`signal_watch_only` 每次触发走一次24小时异动响应，人工核对仅对信号分级强/中执行，弱级仅留痕（v20）。经事件通道激活的（L2×可接受较高估、L3×中性）一律按战术仓候选处理，必经 §10 人工闸门（等同 C 级流程）。

说明：S/A/B/C/D 优先级是"质量×估值"维度，§8.7 信号分级（强/中/弱）是"信号强度"维度，两者正交、输出时并列。仅满足有效放量上涨、未突破前高或关键均线的，仍为 `buy_candidate`（信号分级=中或弱），按 §8.11 的中短期量价强势定级排序。脚本的 S 级判定含前高突破、日线多头、长期趋势确认三项；相对强度确认暂由人工在 §10 核对时补充判定。

### 8.11 买入候选的强势跟踪

`buy_candidate` 一旦产生，必须每个交易日实时跟踪其股价与量价走势；强势程度**仅**由中期和短期量价走势判断，不引入基本面、前高位置等其它维度。

- **短期量价（约 5-10 日）**：close 相对 MA5/MA10，5 日涨幅，vol_5d_ratio_20，近 5-10 日相对沪深300与行业指数的强弱。
- **中期量价（约 20-60 日）**：close 相对 MA20/MA60，20-60 日涨幅，量能趋势（vol_ma5/vol_ma20/vol_ma60 排列），近 20-60 日相对强度。

| 强势 | 含义 | 处理 |
| --- | --- | --- |
| `strong` | 中短期量价同向走强、量价配合健康 | 维持买入候选，按 §8.10 优先级 |
| `neutral` | 中短期走平或量价背离 | 保持观察，等待再次放量上涨 |
| `weakening` | 中短期转弱、放量下跌或缩量阴跌 | 降级，必要时移出买入候选 |

强势跟踪只用量价，不构成买入命令；买入前仍需 §10 人工核对。

### 8.12 市场状态判定

每日扫描时由脚本用沪深300指数计算市场状态，写入 `daily_buy_candidates.csv` 的 `market_state` 字段并随运行输出打印，扫描日志条目头部（§9.2）照抄：

```text
强势 = 沪深300收盘 > MA250 且 MA250 较20日前上行
弱势 = 沪深300收盘 < MA250 且 MA250 较20日前下行
震荡 = 其余情形
```

市场状态不改变信号分级本身，只调整默认操作偏向：弱势市场下各档偏向下调一档（强→可小仓试探或等确认，中/弱→等确认）。总仓位控制遵循个人投资体系 §3。

## 9. 每日例行扫描与输出

### 9.1 组合流程（每交易日一次，支持盘中发起）

用户发起"当日的每日扫描"时按固定顺序执行——**先持仓、后池**：

1. **持仓健康度**：运行 `scan_holdings_sell_signals.py --as-of 当日`（读取账户快照），按 §14 状态字与 §9.2 红黄绿映射汇总每只持仓的健康度、当日量价特征与需要的动作。
2. **池内关注对象**：运行 `screen_daily_volume_price_signals.py --as-of 当日 --review-queue data/interim/a_share_report_update_queue.csv`，从输出中筛选 `buy_candidate`（按 §8.10 优先级 × §8.7 信号分级排序）与 `signal_watch_only`，逐只写出关注原因。
3. **输出**：按 §9.2 格式先以回复形式给出，再将同一内容置顶写入 §9.4 扫描日志。两张 CSV 与决策日志照常由脚本生成（v25 起脚本不再生成独立 tracker 阅读版）；扫描日志是唯一每日阅读文件，不替代机器记录。
4. **盘中口径（未收盘发起时）**：
   - 头部标注"盘中截至HH:MM"；当日 bar 为未完成 bar，"收盘价"实为最新价。
   - 量比类指标按已成交量计算，属系统性低估的**下限口径**；可附"已成交量 ÷ 已交易时间占比"的线性外推估算作参考，须标注为估算。
   - 信号成立与否的正式判定以收盘数据为准，盘中判定一律标注"待收盘定型"。例外：割肉价触及本就以当日价判定（§14 Tier-0），盘中触及即为最终动作信号，不等收盘。
   - **收盘定型**：仅对盘中"已触发或接近触发"的标的（割肉价、强/中级信号、趋势保护线临界）在收盘后复核；结论有实质变化的，在当日日志条目内追加"收盘更新："行，无变化不追加，不要求全池重扫。
5. 同一交易日重复发起（如盘中一次、收盘后一次）时，更新当日日志条目，不新增第二条。

### 9.2 组合摘要格式（回复与日志条目共用）

```text
## 每日扫描 YYYY-MM-DD（口径：盘中截至HH:MM / 收盘）

市场状态：…（§8.12）｜账户回撤状态：…（净资产距 -8% 去杠杆线的剩余空间）

### 一、持仓健康度扫描
总览：红X / 黄Y / 绿Z（共N只）
- 红｜名称（代码）｜状态字｜现价（涨跌%）｜距割肉价%｜当日必须动作
- 黄｜名称（代码）｜状态字｜现价（涨跌%）｜距割肉价%｜关注点与建议
- 绿｜名称（代码）｜现价（涨跌%）｜距割肉价%（多只可并列；有加仓/止盈资格的单独点名）

### 二、估值合格池扫描（关注对象）
- 名称（代码）｜优先级×信号分级｜层级×估值档｜关注原因：触发形态+量价要点+当日催化｜下一步（§10闸门核对 / 等确认 / 等回踩）
- watch_only 触发（可见不可买）：名称（代码）｜信号｜走24小时异动响应 + §7.4复核
（今日无触发时写明扫描只数与"无有效信号"，可附 wait_confirmation 名单）

### 三、今日待办（无则省略本节）
- 需要人工核对或决定的事项（§10 闸门、收盘定型复核、割肉价上调确认等）

收盘更新：（仅盘中口径条目定型后有实质变化时追加）
```

健康度分组只是 §14 状态字的**展示映射**，不引入新的动作阈值：

| 分组 | 对应 §14 状态 |
| --- | --- |
| 红（当日必须处理） | `stop_loss_sell`（盘中触及即最终）、`forced_exit`（Tier-1 成立） |
| 黄（预警/复核） | `risk_alert`、`raise_stop_suggested`、`trend_exit_suggested`、`trend_reduce_half_suggested`、`trend_downgrade_suggested`、`thesis_reduce_suggested`、估值档位恶化、单笔风险超限；现价贴近割肉价的提示（提示口径，不构成动作阈值） |
| 绿（健康） | `hold`；`add_eligible` / `trim_eligible` 归绿但须单独点名资格 |

关注原因必须落在可核对的触发形态与数据上（§8.5/§8.7 术语与量价指标），不得写"感觉强势"类模糊描述；含当日催化的须注明信源等级（公告 / 权威研报 / 媒体报道）。

### 9.3 机器明细（CSV 口径，v25）

原量价/持仓两份 tracker 阅读版自 v25 退役，脚本只写 CSV 与决策日志；逐股全量明细以两张每日 CSV 为准：

- `daily_buy_candidates.csv`：池内全部扫描对象逐行，含 `signal_state`、`priority`、`signal_grade`、`action_bias`、`market_state` 与全部量价指标（§8.4）；`wait_*`、`inactive_watch`、`liquidity_filtered` 等非触发状态只留在 CSV，不进扫描日志。
- `daily_holdings_actions.csv`：每只持仓逐行，含 `holdings_action`、趋势保护判定、盈利离场资格、割肉价上调建议、权重与单笔风险、估值档位刷新。

扫描日志（§9.2）中对信号分级强/中的买入候选与触发动作的持仓，按下列要点行展开；弱级与无触发标的只汇总只数，不逐股展开：

```text
- 优先级×信号分级｜层级×估值档｜触发信号（§8.7 形态）｜量价要点｜相对强度｜当日催化（注明信源等级）｜风险｜操作偏向
```

### 9.4 扫描日志文件

- 路径：`data/processed/000_daily_scan_log.md`，**唯一的每日阅读输出**（v25 简并），**最新条目在最上**：新条目插入文件头部说明与分隔线之下、旧条目之上。
- 条目即 §9.2 格式块（强/中级候选与触发动作持仓按 §9.3 要点行展开），与当日回复内容一致；每交易日一条，重复发起时更新当日条目（§9.1 第 5 条）。
- 日志由大模型在完成组合流程后写入，属阅读汇总：机器记录以 `daily_buy_candidates.csv`、`daily_holdings_actions.csv` 与决策日志为准，日志本身不写决策日志行。

## 10. 买入前闸门（pretrade_decision）

任何 `buy_candidate` 在实际买入前必须完成结构化买入前决策，向 `data/processed/pretrade_decisions.csv` 写入一行，并写决策日志（`workflow_stage = pretrade_decision`）。闸门与个人投资体系 §9.1 通用买入清单一一对应：**任一字段缺失或不达标，状态只能停留在 `buy_candidate`，不得建仓**。

### 10.1 闸门清单（全部满足才能买）

买入资格项（对应个人体系 §9.1）：

1. 已进入观察池且在核心估值合格池内、不处于 §7.5 买入冻结（L3×中性经 §6.2.1 事件通道激活的视同满足本项）。
2. 已完成研究卡片（引用决策日志/分层/估值结论的 `decision_id` 或文件位置）。
3. thesis 清晰（一句话可复述）。
4. 已写 bear/base/bull 三情景，且完成反向预期拆解（市场已定价什么、预期差在哪）。
5. 赔率达标（按质量层级递严，§6.2.1）：L1/L2 按个人体系默认——base 上涨空间 / bear 下跌空间 > 2（高波动产业链股 >3，稳定现金流公司可 1.5-2）；L3 ≥2.5；L4 ≥3。
6. 有右侧信号（引用当日 `daily_buy_candidates.csv` 信号），或极度低估且有催化剂。
7. 已写明证伪条件（什么情况证明自己错了）。
8. 已设定目标仓位（按 §13 口径：质量分层×估值档决定）与分批计划。
9. 已设定加仓、减仓、清仓条件。
10. 已完成催化剂地图：每条写明具体事件、预计时点、影响程度、是否已被定价；写不出可追踪、可验证催化剂的，重新评估买入逻辑。
11. 已对前 2-3 个高概率高影响风险写好预设应对（“触发条件 -> 动作”），建仓时并入持仓清单的 `bear_case` 与 `exit_conditions`。
12. 已核对买入后组合暴露：单票、产业链、大行业上限，以及与现有持仓共享的隐藏 beta。
13. 单笔风险校验通过：首批金额 × 止损距离 ≤ 账户净资产 1.5%（§13）。

当日事实核对：

14. 当日公告是否有真实催化；是否存在澄清公告、监管问询、减持、解禁、融资、财务异常。
15. 股价是否已经过度延伸（§8.8）。
16. 同行业是否存在更强替代标的。

## 11. 固定风险过滤器

以下股票不进入每日买入候选：

1. 不在核心估值合格池的可买层（core/tactical）中；watch_only 层与池外股票的量价信号仅留痕并走复核（§8.9 `signal_watch_only`），不列买入候选。
2. 估值档位为高估或无法估值。
3. 当前业务thesis被证伪。
4. 财务造假、审计异常、控股股东资金占用、重大治理风险。
5. 当日极端放量长上影且未收在高位。
6. 连续暴涨后远离20日线，且无明确业绩或订单上修。
7. 只因题材、小作文、传闻上涨，缺少公告、财报、订单、客户验证或权威研报支持。
8. 20日均成交额 `amount_ma20` 低于5000万元：流动性不足，脚本判定后输出 `liquidity_filtered`，不列入买入候选，仅提示。
9. 处于 §7.5 复核期买入冻结（`review_pending`）：财报或重大事件复核未完成前不列入买入候选，输出 `buy_blocked_review_pending`。

## 12. 历史回放校准

### 12.1 校准目的

在脚本正式作为每日流程使用前，必须验证该流程能识别历史上的高质量右侧启动机会。

### 12.2 外部回放案例

具体回放案例不写入本流程文件。回放案例应作为外部验证集单独维护，避免把用于校准的历史样本固化为流程规则。

每个外部回放案例至少记录：

1. 回放股票代码。
2. 回放日期。
3. 当时已公开的公司资料、财报、公告、研报和行业数据。
4. 当时已发生的价格和成交量数据。
5. 该案例用于验证的流程环节：质量分层、估值排雷、量价触发或卖出规则。
6. 成功标准。

### 12.3 回放脚本接口

```bash
python3 scripts/backtest_signal_replay.py \
  --as-of YYYY-MM-DD \
  --symbols CODE1,CODE2 \
  --workflow-version a-share-selection-operation-v1 \
  --input data/processed/a_share_core_valuation_pool.csv \
  --output data/interim/replay_external_validation_YYYYMMDD.csv
```

说明：`backtest_signal_replay.py` 复用每日量价扫描规则，用于验证历史日期前的价格和成交量信号。完整历史回放必须提供当时已公开信息构造的 `--input` 核心估值合格池；如果使用当前核心估值合格池，则只能验证价格信号，不能证明当时已经满足质量和估值前置条件。回放输入含 watch_only 层时，成功标准须区分「池内直接触发」与「watch_only 可见→复核→入池后触发」两种捕获路径（v20）。

### 12.4 回放数据限制

回放只能使用回放日期之前已经公开的信息：

1. 当时已经披露的年报、季报、公告、业绩预告。
2. 当时已经发布的权威研报。
3. 当时已经发生的成交量、价格、指数数据。
4. 当时已经可见的行业政策和产业数据。

不得使用回放日期之后的财报、研报、订单、股价走势或事后总结。

硬约束：回放使用的每条证据必须满足 `available_at <= 回放日`，其中 `available_at` 是公开可得日（公告日/发布日），**不是报告期末**。财务指标表以 `latest_report_notice_date`（公告日）为准；公告日缺失的数据不得直接用于回放，只能按“报告期末 + 法定披露时限”做保守估计并在回放记录中显式标注，或弃用该字段。

### 12.5 回放失败后的固定调整

如果任一外部回放案例失败，按以下规则调整流程：

1. 对产业链爆发股和战略能力型公司，降低当前利润在估值中的权重。
2. 对处于研发、产能爬坡、商业模式切换、行业爆发前夜的公司，强制使用归一化利润和未来2-3年正常化利润。
3. 对强技术壁垒、客户验证、产能瓶颈、行业渗透率快速提升的公司，即使当前PE偏高，也允许进入核心估值合格池。
4. 对低利润或亏损公司，必须区分“真实能力尚未兑现”和“商业模式无效”。
5. 如果公司通过质量分层和估值排雷，但量价未触发，则保持核心估值合格池，不强行买入。

## 13. 固定仓位动作

每日量价扫描只输出操作偏向：

| 操作偏向 | 执行含义 |
| --- | --- |
| 可建目标仓位1/3 | 允许建立计划仓位的第一笔 |
| 等回踩 | 等待回踩20日线、60日线或突破平台不破 |
| 等确认 | 等待3-5个交易日持续承接 |
| 仅观察 | 不买入，只继续追踪 |
| 放弃 | 触发信号无效或估值/业务不合格 |

目标仓位与建仓金额统一口径（贯通个人体系 §7/§9 与 §14 建仓金额规则）：

1. **仓位基准（v20）**：单票与合计上限按总资产（净资产+融资负债）计；风险预算（单笔1.5%、回撤梯-8%/-12%/-20%）仍按净资产口径。有杠杆时总资产口径仓位对净资产的真实暴露按杠杆倍数放大（当前约1.77x），下跌中同一仓位的净资产占比自动升高，回撤梯必须严格执行。
2. **单票上限（v20，按买入类型二分）**：低估优质仓——L1/L2×低估/较低估且有§8右侧信号→单票≤10%总资产；突破/事件仓——池内其它可买组合（L1×中性/可接受较高估、L2×中性、L3×低估/较低估）出现§8信号，或L2×可接受较高估、L3×中性经事件通道（§6.2.1 执行说明4）→单票≤5%总资产；L4×低估→单票≤2.5%总资产、不加仓。L1的15%-25%核心带自v20暂停使用，待体系连续稳定运行两个季度后复核启用。
3. **一笔建仓（v20）**：机会确认后一笔建满目标仓位，不再强制1/3分批；前置不可省——下单前完成§10闸门全项、给定割肉价、通过单笔风险校验：买入金额×止损距离≤净资产×1.5%（止损距离=(买入价-割肉价)/买入价；10%仓位隐含止损距离≤约8.5%，止损需更宽时按公式缩仓，不得直接放行）。纯左侧（无§8信号）的低估买入首笔限5%，右侧确认后方可按下方加仓条件补足至10%。
4. **合计上限不变**：战术/突破仓合计≤15%、同一产业链≤40%、同一大行业≤60%、单票绝对红线30%（存量）；build_amount 金额制自v20退役（§14），持仓清单 `build_amount_cny` 改记实际建仓金额。

加仓必须满足：

1. 买入后趋势继续成立。
2. 回踩不破平台或均线。
3. 财报、订单、利润率、行业数据或盈利预测至少一项继续验证。
4. 当前估值仍未进入高估档。

退出全部遵循阶段五（§14），不单列统一技术退出条件：无条件清仓只由割肉价与硬离场例外决定；趋势离场按保护线档位分档复核（`daily`=MA60 / `weekly`=MA150 / `monthly`=MA250）；未达保护线级别的技术走弱信号（右侧失败跌回平台、跌破MA60后3-5日不收回、高位放量滞涨后缩量阴跌）对 `daily` 档即为破位、对 `weekly`/`monthly` 档仅作上调割肉价与风险预警依据；thesis或估值证伪与重大治理风险按 §14 硬离场例外处理。

### 建仓期状态（2026-07-12 用户声明，撤销前有效）

当前持仓并非按本流程建立。自 2026-07-12 起账户处于**建仓期**，持仓清单以 `position_status` 字段逐只定级，目标是向流程合规状态收敛：

| `position_status` | 含义 | 锁定期 | 割肉价 | 调整权限 |
| --- | --- | --- | --- | --- |
| `qualified` | 层级×估值组合在池内可买层、权重在 §13 上限带内、趋势保护线未破位 | 正常执行（自 `entry_date` 起 3 个月） | Tier-0 硬底，触及当日无条件清仓 | 按 §14 正常规则 |

执行约定：

1. 定级依据 = §6.2.1 分层×估值准入矩阵的可买身份 × §13 单票上限 × 趋势保护线状态；估值池更新或持仓重大调整后复核定级，定级设定与变更写决策日志（`workflow_stage = position_status_review`）。

3. 调整以向 §13 上限带收敛为目标：减仓、清仓、上调割肉价等降风险方向随时可执行；**买入侧不放宽**——加仓仍须满足 §13 加仓条件，新建仓仍须过 §10 闸门。
4. 卖出扫描脚本不读取 `position_status`，其锁定/割肉标注对 `pending_adjustment` 行仅供参考，由扫描日志条目按本节口径改写。
5. 用户宣布建仓期结束后：剩余持仓全部转 `qualified`（或清仓离场），删除本节与 `position_status` 相关过渡注释，锁定期与割肉价规则全量恢复。
6. **显式固定声明（2026-07-14 用户声明）**：用户可点名将个别持仓显式固定为 `qualified` 并启用锁定期（自该持仓 `entry_date` 起 3 个月），本声明优先于第 1 条的自动定级依据——权重超 §13 上限不阻断定级，以 `risk_alert` 持续跟踪且禁止加仓；趋势破位项按 §14 锁定期交互处理（只出 `risk_alert` 与上调割肉价建议）。固定后割肉价恢复 Tier-0 硬底；割肉价缺失的行必须尽快由用户补定，补定前该行无 Tier-0 保护（仅 Tier-1 可破锁），每日扫描须置顶提示。首批适用：药明康德（锁至 2026-09-24）、比亚迪、美的集团（均锁至 2026-10-07）、贵州茅台（锁至 2026-10-13，割肉价待补）。

### 建仓金额规则

1. v20 起单票建仓金额上限由 §13 的百分比口径决定（低估优质仓≤10%总资产、突破/事件仓≤5%、L4≤2.5%），一笔建仓；总资产 = 净资产 + 融资负债，原则上不持有现金。
2. 建仓前须通过 §10 买入前闸门与 1.5%（净资产口径）单笔风险校验。
3. 原 build_amount 金额制（25万起、总资产达 build_amount*20 翻倍）自 v20 退役；持仓清单 `build_amount_cny` 字段改记实际建仓金额，历史行含义不变。

### 割肉（止损）规则

1. 买入时必须在持仓清单写入 `stop_loss_price`（割肉价，由用户给定）。
2. 当日价 <= `stop_loss_price` 时，无条件清仓，不受 3 个月锁定限制；**当日或次日开盘执行，不得以“等待 1-3 日确认”延迟**（见退出优先级矩阵 Tier-0）。
3. 在触及割肉价之前，若持有时间 < 3 个月，禁止任何卖出；但触发退出优先级矩阵 Tier-1 硬证伪时除外（见下）。
   - **建仓期（2026-07-12 用户声明，撤销前有效）**：按本阶段开头「建仓期状态」执行——`pending_adjustment` 持仓无锁定、割肉价仅为参考（触及当日须显式复核留痕）；`qualified` 持仓正常适用本条与 Tier-0。原 2026-07-11 建仓期豁免注并入本条。用户宣布建仓期结束后恢复并删除本注。
4. `stop_loss_price` 只能上调、不能下调；每次上调记录 `stop_loss_last_raised_date`。
5. 上调后的 `stop_loss_price` 不得高于当日 MA120，避免用过高的割肉价作为卖出借口；例外（v20）：`trend_protection_level = daily` 的战术/事件仓按启动结构管理割肉价（启动平台、启动日低点、缩量回踩低点），不受 MA120 上限约束。

### 退出优先级矩阵（锁定期与卖出规则的统一口径）

与个人投资体系 §11 的衔接：thesis/估值/风险各类卖出信号按以下三级处理，级别决定能否打破 3 个月锁定。

| 级别 | 事件 | 锁定期内 | 锁定期外 |
| --- | --- | --- | --- |
| **Tier-0 无条件** | 触及割肉价 | 立即清仓（`stop_loss_sell`） | 同左 |
| **Tier-1 硬证伪（可破锁清仓，`forced_exit`）** | ① 一票否决：财务造假、控股股东资金占用、审计异常等确凿治理/财务问题；② 公司重大突发事件；③ 季报披露后利润严重不及预期；④ **经核实的结构性 thesis 证伪**：核心大订单取消或重大客户流失被公告/权威信源确认、技术路线被颠覆致产品被实际替代、政策逻辑反转使 thesis 不再成立 | 大模型判定 + 卖出前人工核对确认后立即清仓 | 同左 |
| **Tier-2 软走弱（不可破锁）** | thesis 走弱但未坐实（订单延迟未取消、利润率未如期改善、竞争加剧、验证进度落后）；估值过热（价格已反映 bull case） | 只允许上调割肉价 + 风险预警，不得卖出 | thesis 走弱：可减仓至半仓复核（`thesis_reduce_suggested`），不受浮盈档位限制；估值过热：属止盈性质，减仓受“盈利后离场规则”上限约束 |

判定纪律：Tier-1 第④类必须以公告、监管文书或多个权威信源核实为准，“担心”“传闻”“股价下跌”一律按 Tier-2 处理；Tier-2 事件累积不升级为 Tier-1，但可作为持续上调割肉价的依据，由割肉价机制完成实际离场。

### 除权除息处理

分红、送转、配股的除权除息日，必须在当日扫描前按除权除息因子机械调整持仓清单中的价格与股数字段：`stop_loss_price`、`cost_basis`、`launch_platform_price` 按因子调整；送转/配股同时调整 `initial_shares`、`current_shares`。该调整是价格口径换算，**不属于"下调割肉价"**；浮盈与盈利离场档位按调整后口径计算。K线使用前复权数据，未调整的绝对价格字段会在除权日引发误触发（现价机械下跳击穿割肉价）或浮盈失真。持仓清单为手工维护，调整由维护者完成，并在 `notes` 记录调整因子与日期。

### 趋势保护线（软性减仓/复核，非无条件清仓）

割肉价仍是唯一的无条件清仓硬底；趋势保护线是"多头趋势消失"的分档响应，触发后输出**减仓/退出建议**，必须经卖出前人工核对后执行。分档目的：防止把长周期复利仓当波段做——历史上强势核心股（如2020年宁德时代、2025-2026中际旭创）主升途中会反复跌破 MA20/60 甚至回踩 MA120，但极少连续多日深跌于 MA150/MA250 之下，紧线止损只会增加调仓成本。

三档保护线全部用日线均线与日线收盘判定。v12 起废弃周/月已完成周期收盘判定：周期末单点采样噪声大（月末恰逢急跌即触发、月中深跌月末拉回则无信号），且月线档最坏延迟接近一个整月；日线持续性判定的最坏响应约 10 个交易日，抗噪能力更强。

| 持仓类型（`trend_protection_level`，留空按 strategy_tag 映射） | 保护线 | 跌破定义 | 触发动作 |
| --- | --- | --- | --- |
| `daily`（战术/事件仓，仅显式标注） | MA60 或突破平台 | 连续 3 日收盘低于线，且最新收盘距线跌幅 > 3% | `trend_exit_suggested`：退出复核 |
| `weekly`（B/C/D/E：烟蒂、成长、产业链爆发、落难白马） | MA150（约30周） | 连续 5 日收盘低于线，且最新收盘距线跌幅 > 3% | `trend_reduce_half_suggested`：减半复核 |
| `monthly`（A/F/G：复利、资源、股东回报） | MA250（年线，约12个月） | 连续 10 日收盘低于线，且最新收盘距线跌幅 > 5% | `trend_downgrade_suggested`：降档减仓复核（核心降准核心） |

标准化跌破与站稳（默认参数，如需调整先改本文件）：

1. 统一规则：跌破 = 连续 N 个交易日收盘低于保护线，且最新收盘距线跌幅 > X%（N/X 按上表档位取 3日/3%、5日/3%、10日/5%）；站稳 = 连续 N 个交易日收盘站上保护线，且保护线走平或上翘。
2. 一律用收盘价，不用盘中最低价；横盘期间收盘在线上下反复但未满足上述定义的，不算跌破。

与割肉价的关系：割肉价上调至接近 MA120 后会先于 MA150/MA250 档触发，趋势保护线主要对割肉价留得较深的持仓起兜底作用。两层并存、职责不同：割肉价是可紧可松的无条件硬底，趋势保护线是多头趋势消失后的软性分档复核。

锁定期交互：3 个月锁定期内趋势破位**不输出卖出建议**，只输出 `risk_alert` 并给出上调割肉价建议。

### 盈利后离场规则

仅当持有时间 > 3 个月且当前浮盈为正时适用。基准为 `initial_shares`，按浮盈比例给出“允许累计卖出上限”；默认仍是持有——盈利本身不是卖出理由。

| 浮盈（相对建仓成本） | 累计允许卖出上限（占 `initial_shares`） |
| --- | ---: |
| >= +30% | 10% |
| >= +50% | 20% |
| >= +100% | 35% |
| >= +200% | 50% |

到 +200% 累计卖出 50% 时，已基本收回本金。`cumulative_trim_pct` 不得超过当前浮盈档对应的上限。这些是允许卖出的上限，不是强制卖出指令。

脚本 `scripts/scan_holdings_sell_signals.py` 负责确定性部分：割肉触及、3 个月锁定、趋势保护线分档判定（MA60/MA150/MA250，日线统一规则）、盈利离场资格、割肉价上调建议（不超 MA120）、建仓金额与单票权重、持仓估值档位刷新（对照核心池，档位恶化或已出池输出 `risk_alert`）、账户回撤状态与杠杆/担保预警（读取账户快照最新行）、单笔风险校验（权重×止损距离对净资产的比例，超 1.5% 提示上调割肉价）。退出优先级矩阵 Tier-1 硬证伪与是否上调割肉价由大模型判定。`--valuation-pool` 与 `--account-snapshot` 缺省使用固定产物路径，文件缺失时降级运行并在输出头部标注。

### 脚本可判定的卖出与风险触发

按以下优先级处理每个持仓：

1. 硬证伪离场（最高优先级）：退出优先级矩阵 Tier-1 由大模型判定成立 → `forced_exit`，立即清仓，打破 3 个月锁定。
2. 割肉清仓（无条件，Tier-0）：当日价 <= `stop_loss_price` → `stop_loss_sell`，全部清仓，不受 3 个月锁定限制。
3. 3 个月锁定：未触及上述例外或割肉价，且持有 < 3 个月 → 一律 `hold`，不输出任何卖出；锁定期内趋势破位只输出 `risk_alert` + 上调割肉价建议。
4. 趋势保护触发：按持仓的保护线档位与标准化跌破定义判定 → `trend_exit_suggested` / `trend_reduce_half_suggested` / `trend_downgrade_suggested`（均为复核建议，非自动卖出）。
5. 盈利离场资格：持有 > 3 个月且浮盈为正 → 按“盈利后离场规则”计算当前允许累计卖出上限；若 `cumulative_trim_pct` 低于该上限 → `trim_eligible`，给出本次最多可卖比例（默认仍可选择持有）。
6. 割肉价上调建议：若现价显著高于 `stop_loss_price`，且当日 MA120 高于现 `stop_loss_price` → `raise_stop_suggested`，建议上调到不超过 MA120。

风险预警（仅提示，不直接触发卖出）：

1. 单票 > 30%、同一产业链 > 40%、同一大行业 > 60%、战术持仓合计 > 15%（阈值与个人投资体系 §2.1 同步）。
2. 账户自峰值回撤 -8% / -12% / -20% → 去杠杆 / 黄色警告 / 红色警告（与个人投资体系 §13.1 同步；脚本读取账户快照计算）。
3. 未达到趋势保护线级别的技术走弱信号（高位放量滞涨、缩量阴跌、跌破 MA60 且 3-5 日未收回等）→ 作为上调割肉价的依据，不直接卖出。
4. 单笔风险超限：持仓权重 × 当前止损距离 > 净资产 1.5%（个人体系 §13.2 的持仓监控版）→ 提示上调割肉价收窄风险，不直接卖出。
5. 持仓估值档位恶化：对照最新核心池，档位降为高估/无法估值或已出池 → 提示复核（锁定期外结合盈利离场上限处理）。

### 大模型判定的卖出与风险信号

硬证伪离场（判定成立即 `forced_exit`，立即清仓，打破 3 个月锁定）：四类清单见「退出优先级矩阵」Tier-1，不再重复；第④类结构性 thesis 证伪必须以公告/监管文书/多个权威信源核实为准。

Tier-2 软信号按矩阵处理（锁定期内只上调割肉价与风险预警；锁定期外 thesis 走弱可减半复核 `thesis_reduce_suggested`、估值过热受盈利离场上限约束）：

1. thesis 走弱（未坐实）：订单延迟未取消、客户验证进度落后、毛利未如期改善、竞争加剧、技术路线或政策逻辑边际变化、管理层口径与披露数据出现张力——坐实即升级 Tier-1。
2. 估值过热：当前价已反映 bull case、未来 2-3 年利润已被定价、股价远快于盈利上修。
3. 催化剂对照：对照买入时的催化剂地图（§10.1 第 10 条），已兑现的按催化兑现重估，无下一轮催化或盈利上修则建议减仓；超期未兑现的作为时间维度走弱信号。连同机会成本变化，作为是否上调割肉价或调整观察的参考。

### 卖出前人工核对

任何 `stop_loss_sell`、`forced_exit`、`trim_eligible`、`thesis_reduce_suggested`、`trend_exit_suggested`、`trend_reduce_half_suggested`、`trend_downgrade_suggested` 在实际执行前必须核对：

1. 当日是否有澄清公告或被误读的利空（含触发所依据的数据是否有误）。
2. 是否处于不可交易状态（停牌、一字跌停）。
3. **仅对趋势保护线与 Tier-2 类软信号**：触发是否为单日噪声，是否需等待1-3个交易日确认；`stop_loss_sell` 与 `forced_exit` 不适用本条，核对仅限第 1、2 项，通过即当日/次日执行，不得延迟。
4. 减仓比例是否符合盈利后离场规则的允许上限（`thesis_reduce_suggested` 除外）。
5. 卖出后若反向走强的再买回条件。
6. 执行后按 §10.3 回写持仓清单并记 `execution_record`。

### 与买入侧的衔接

1. 阶段四产生 `buy_candidate` 并实际建仓后，必须把该持仓写入 `a_share_holdings.csv`，记录策略标签、`build_amount_cny`、`initial_shares`、`cost_basis`、`stop_loss_price`、突破平台价、bear case与退出条件。
2. `a_share_holdings.csv` 与买入候选相互独立：买入候选是池内信号，持仓清单是已建仓事实。
3. 每个持仓动作结论写入决策日志，`workflow_stage = holdings_sell_scan` 或 `manual_sell_check`。

## 15. 执行方式

本文件是A股选股、估值、量价扫描的主执行规范。后续可以用类似以下指令执行固定流程：

```text
请基于 docs/000_Ashare_workflow.md 执行今天的A股每日量价扫描，并将结果写入固定输出文件和结论日志。
```

执行分工：

1. 能由脚本稳定完成的部分必须用脚本执行，例如证券名单更新、队列生成、核心估值合格池物化、每日量价指标扫描、每日持仓技术与风险触发扫描和历史回放。
2. 需要判断业务质量、护城河、估值模型、研报假设、公告真实性和用户defence的部分，由大模型按本文件中的执行规则和输出字段执行。
3. 大模型执行时不得临时改标准；如发现标准不合理，先修改本文件，再按新版本重新执行。
4. 每个产生结论的脚本输出和每个大模型批次结论都必须写入 `data/processed/a_share_workflow_decision_log.csv`；队列生成脚本只做调度，不单独写结论日志。
5. 每日量价扫描不是买入命令。买入前必须通过 §10 买入前闸门（pretrade_decision 全字段齐备且 `approved`），成交后按 §10.3 回写。
6. 每日持仓卖出扫描不是卖出命令。无条件清仓只由割肉价（Tier-0）与退出优先级矩阵 Tier-1 硬证伪决定；趋势保护线触发分档减仓/退出建议、Tier-2 软走弱减仓、止盈受盈利后离场上限约束，均须完成阶段五（§14）的卖出前人工核对后执行并回写。
7. **调仓时间窗（2026-07-14 用户纪律）**：任何主动调仓（新建仓、加仓、减仓、止盈、清仓）只允许在交易日 14:20 之后执行——尾盘时段当日 bar 接近定型，避免盘中冲动交易与被日内噪声甩出。例外：Tier-0 割肉价触及与 Tier-1 硬证伪不受本窗约束，仍按 §14 触及即执行、不得延迟。盘中扫描产生的动作建议一律标注"14:20 后执行"。

### 15.1 重蹈覆辙拦截（个人体系 §17 的执行入口）

用户指令命中个人投资体系 §17.3 红线清单（R1-R7：杠杆下无信号左侧买入、无证据摊平、口头承诺当证据、下调/取消割肉价、超限集中或加杠杆、失败仓转长期仓、盈利期拆防护层）时，执行代理必须先按 §17.3 拦截协议纠正劝阻，不得直接执行：

1. 答复中引用命中的红线编号与既往病历事实（迈瑞 2025-09：约 240 全仓 → 200/180 无证据摊平 → 约 154 清仓 -36%，期间担保率一度约 128% 逼近强平线）。
2. 给出体系内替代路径（左侧想法 → 个人体系 §9.1 第 5 条催化剂通道 + 本文件 §13 纯左侧首笔 5% 限制；摊平 → 列出所需新增证据清单）。
